# 템플릿 (A09, T09)

SA/TA 공통. 가상 PC의 "설계도"를 관리하는 3개 화면. 템플릿 = Flavor(사양) + Image(OS 이미지) 조합.

**3개 메뉴 모두 컴포넌트 내부에 `isSuperAdmin` 분기 없음** — SA/TA가 완전히 동일한 UI. 차이는 메뉴 접근 권한(RBAC)에서만 발생.

## 3개 메뉴의 관계

```
가상 PC 사양 (A0902, T0902) = Flavor
  "CPU 4코어, 메모리 8GB, 디스크 50GB"
  │
  └─── 조합 ───→ 템플릿 (A0901, T0901)
                   "Windows 10 + 4코어 + 8GB"
  ┌─── 조합 ───┘    │
  │                  └── 가상 PC 그룹/풀에서 이 템플릿 선택
베이스 이미지 (A0903, T0903)   → VM 생성 시 이 스펙으로 생성
  "Windows 10 OS 이미지"
```

### 비유

```
가상 PC 사양 (Flavor) = 자동차 엔진/옵션 스펙
베이스 이미지 = 자동차 모델 (차체)
템플릿 = 완성된 자동차 사양서
  → 이 사양서대로 자동차(VM)를 찍어냄
```

### 생성 순서

```
1. [인프라팀/SA] 베이스 이미지 등록
     → OpenStack Glance에 OS 이미지 업로드
     → /initialized 마법사에서 동기화

2. [SA/TA] 가상 PC 사양 생성
     → Flavor 정의 (CPU/RAM/디스크)
     → OpenStack에 Flavor 동기화

3. [SA/TA] 템플릿 생성
     → Flavor + Image 선택하여 조합

4. [TA] 가상 PC 그룹 생성 시 템플릿 선택
     → 풀의 VM들이 이 템플릿 스펙으로 생성
```

## 가상 PC 사양 (A0902, T0902)

경로: /template/virtual-spec

컴포넌트: views/template/VirtualSpec.vue / VirtualSpecDetail.vue

OpenStack Flavor 관리. CRUD.

### 테이블 컬럼

| 컬럼 | 필드 | 설명 |
|------|------|------|
| 유형 | `flavor_bas_yn` | Y: 기본(삭제 불가) / N: 일반 |
| 사양명 | `flavor_nm` | |
| CPU | `vcpu_cnt` | 코어 수 |
| 메모리 | `vmm_capa` | 입력: MB 단위, 표시: MB/GB/TB 자동 변환 |
| 디스크 | `vhd_capa` | GB |
| TPM | `tpm_yn` | TPM 지원 여부 (보안 칩) |
| 생성자 | `reg_conn_id` | |
| 생성일시 | `reg_ts` | |

### 메모리 단위 처리 (IAADEV-5500)

- 입력(등록/수정): MB 단위로 통일 (OpenStack과 동일)
- 표시(조회): `formatExprAllUnitCapa()` 함수로 자동 변환 (1024 MB → 1 GB, 1048576 MB → 1 TB)

### VirtualSpecDetail 필드 상태

상세 폼의 대부분 필드는 `:disabled="true"`로 SA/TA 모두 편집 불가 (조회용). Flavor ID는 신규 등록 시만 편집 가능.

### 골든 이미지 생성 (A090302, T090302)

경로: /template/golden-image/create

컴포넌트: views/template/goldenImage/GoldenImageCreate.vue

기존 VM에서 OS 이미지를 추출하여 새 베이스 이미지로 등록. `isSuperAdmin` 분기 없음.

TA 메뉴 트리에도 T090302가 존재. 실제 TA 접근 가능 여부는 RBAC 설정에 따라 달라짐.

```
골든 이미지 생성 흐름:
  1. 생성용 가상 PC 목록 조회 (3초 폴링)
  2. VM 선택 → 접속하여 OS 커스터마이징
  3. "배포" 클릭 → VM의 현재 디스크를 Glance 이미지로 변환
  4. 새 베이스 이미지로 등록 → 이후 템플릿에서 선택 가능
```

#### 버튼 disabled 조건 — 행 데이터의 상태값으로 결정 (SA/TA 동일)

| 버튼 | enabled 조건 | 설명 |
|------|------------|------|
| 생성 | `create_btn == 'Y'` | 이미지 생성 시작 |
| 접속 | `conn_btn == 'Y'` | VM 콘솔 접속 |
| 배포 | `convert_btn == 'Y'` | 이미지 배포 |
| 초기화 | `clear_btn == 'Y'` | 작업 초기화 |

| 동작 | 경로 |
|------|------|
| 생성용 VM 목록 | `GET /v1/resource/images/vpc` |
| 상태 조회 | `GET /v1/resource/images/status/{img_id}` |
| 전원 ON/OFF | `PUT /v1/resource/images/{img_id}/power_on` or `power_off` |
| 초기화 | `PUT /v1/resource/images/clear/{img_id}` |
| 삭제 | `DELETE /v1/resource/images/delete/{img_id}` |

## 전체 연결 — 템플릿에서 VM까지

```
[SA/TA] Flavor 생성 (CPU 4, RAM 8GB, Disk 50GB)
  +
[인프라/SA] Image 등록 (Windows 10 골든 이미지)
  ↓
[SA/TA] 템플릿 생성 ("Win10-Standard" = Flavor + Image)
  ↓
[TA] 가상 PC 그룹 생성 → 템플릿 선택
  ↓
[TA] 풀 생성 → 서브넷/정책 지정
  ↓
[TA] 사용자에게 VM 할당 → 풀에서 VM 생성
  → CPU 4코어, RAM 8GB, Disk 50GB, Windows 10
  → 지정된 서브넷에 IP 할당
  → 지정된 Zone의 호스트에 배치
```
