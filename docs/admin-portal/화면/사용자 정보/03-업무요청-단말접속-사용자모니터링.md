# 업무 요청 / 단말 접속 / 사용자 모니터링

사용자 정보 하위 메뉴 중 관련성이 높거나 내용이 적은 3개 메뉴 통합 정리.

모두 TA 전용 (SA 메뉴에 없음). `isSuperAdmin` 분기 없음.

---

## 업무 요청 (T0203)

경로: /user/job-request

컴포넌트: views/userInfo/JobRequest.vue

상세: views/userInfo/JobRequestDetail.vue

user-portal에서 사용자가 신청한 업무 요청을 TA가 조회·처리.

### 사용자 지원 > 업무 처리 요청 (T0302)과의 관계

| 항목 | 업무 요청 (T0203) | 업무 처리 요청 (T0302) |
|------|------------------|---------------------|
| 위치 | 사용자 정보 하위 | 사용자 지원 하위 |
| 데이터 | 동일 API (`/v1/user/work/request`) | 동일 |
| 관점 | 사용자 단위 — 특정 사용자의 요청 이력 | 전체 — 모든 사용자의 요청 목록 |
| 상세 폼 | `JobRequestDetail.vue` | `JobWorkRequestDetail.vue` (다른 컴포넌트) |

### UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조.

### 필터

| 필터 | 필드 | 설명 |
|------|------|------|
| 진행상태 | `pgrs_sts_cd` | 드롭다운 (코드: J001) |
| 요청채널 | `req_ch_cd` | 드롭다운 (코드: J004) |
| 요청유형 | `usr_req_div_cd` | 드롭다운 (코드: J003) |

### 테이블 컬럼

| 컬럼 | 필드 | 설명 |
|------|------|------|
| 상태 | `pgrs_sts_cd` | 상태 아이콘 |
| 요청채널 | `req_ch_cd_nm` | — |
| 요청유형 | `usr_req_div_cd_nm` | — |
| 요청자ID | `req_acct_conn_id` | — |
| 요청일시 | `demd_tm` | — |
| 처리자ID | `had_acct_conn_id` | — |
| 처리일시 | `had_tm` | — |

### 행 클릭 → 상세 (JobRequestDetail)

| 항목 | 필드 | 설명 |
|------|------|------|
| 처리상태 | `pgrs_sts_cd` | 드롭다운. ↓ **`J001A`(승인) 상태이면 disabled** |
| 요청채널 | `req_ch_cd_nm` | 읽기 전용 |
| 요청자ID | `req_acct_conn_id` | 읽기 전용 |
| 대상사용자ID | `tgt_acct_conn_id` | 읽기 전용 (없으면 '-') |
| 개인PC ID | `tgt_vm_nm` | 읽기 전용 (없으면 '-') |
| 요청유형 | `usr_req_div_cd_nm` | 읽기 전용 + `상세보기` 버튼 |

`상세보기` 버튼 — ↓ **J003ERR, J003DAR 제외 시 노출**. 요청 유형에 따른 상세 정보 모달.

### API

| 동작 | API | 명세 |
|------|-----|------|
| 요청 목록 | `GET /v1/user/work/request` | [명세](../../api/user/work/01-work.md) |

## 단말 접속 (T0204)

경로: /user/terminal-access

컴포넌트: views/userInfo/TerminalAccessList.vue

상세: views/userInfo/TerminalAccessDetail.vue

사용자별 접속 가능 단말(PC, Thin Client 등) 관리. 단말 식별 정보 등록/수정/삭제.

### 연관 관계

```
[테넌트 설정] 단말 접속 제한 (viwr_con_plcy_yn) → 테넌트 레벨 ON/OFF
     │
     ▼
[사용자 그룹] (T0201) 단말 접속 사용여부 → 그룹 레벨 ON/OFF
     │
     ▼
[단말 접속] (T0204) → 개별 사용자의 단말 식별 정보 등록/관리
```

### UI 레이아웃

Split Panel (상하 분할). 상단 리스트 + 하단 상세.

### 테이블 컬럼

| 컬럼 | 필드 | 설명 |
|------|------|------|
| 사용자ID | `acct_conn_id` | — |
| 사용자명 | `acct_nm` | — |
| 사용자 그룹 | `usr_grp_nm` | — |
| ADB 옵션 여부 | `viwr_con_plcy_yn` | Y/N |
| 최대 단말 수량 | `viwr_con_cnt` | — |
| 현재 단말 수량 | `acct_dvc_ident_count` | — |
| 최종수정일시 | `mod_ts` | — |

### 행 클릭 → 상세 (TerminalAccessDetail)

#### 사용자 정보 (읽기 전용)

사용자ID, 사용자명, 사용자 그룹, ADB 옵션 여부, 최대 단말 수량 표시.

#### 단말 접속 정보 테이블

등록된 단말 목록:

| 컬럼 | 필드 | 설명 |
|------|------|------|
| Client OS | `clnt_dvc_type` | 클라이언트 OS 타입 |
| 단말명 | `dvc_nm` | **필수** |
| 단말ID | `dvc_id` | 선택 |
| MAC 주소 | `mac_addr` | 선택 |
| IP 주소 | `ip_addr` | 선택 |

#### 버튼

| 버튼 | 설명 |
|------|------|
| 단말 등록 | `TerminalAccessDeviceSelect` 모달 → 단말 정보 입력 후 등록 |
| 수정 | `TerminalAccessDeviceModify` 모달 → 선택한 단말 정보 수정 |
| 삭제 | 확인 팝업 → 선택한 단말 삭제 |

### API

| 동작 | API | 명세 |
|------|-----|------|
| 단말 목록 | `GET /v1/user/device-identifier/users` | [명세](../../api/user/device-access/01-device-access.md) |
| 단말 등록 | `POST /v1/user/device-identifier` | [명세](../../api/user/device-access/01-device-access.md) |
| 단말 수정 | `PUT /v1/user/device-identifier/{acctDvcIdentId}` | [명세](../../api/user/device-access/01-device-access.md) |
| 단말 삭제 | `DELETE /v1/user/device-identifier/{acctDvcIdentId}` | [명세](../../api/user/device-access/01-device-access.md) |

## 사용자 모니터링 > 접속 환경 (T020601)

경로: /user/policy/policy-list

컴포넌트: views/userInfo/policy/policyList.vue

사용자의 VDI 접속 환경 요청/응답 로그 조회. 정책 적용 결과 모니터링.

### UI 레이아웃

리스트 단일 화면.

### 필터

| 필터 | 필드 | 설명 |
|------|------|------|
| 전체 | `''` | 기본값 |
| 사용자ID | `acct_conn_id` | 검색 |

### 테이블 컬럼

| 컬럼 | 필드 | 설명 |
|------|------|------|
| 사용자ID | `acct_conn_id` | — |
| VM | `vm_nm` | — |
| 요청 | `request_data` | `보기` 버튼 → 모달 |
| 응답 | `response_data` | `보기` 버튼 → 모달 |
| 결과 | `result` | Y=성공 / N=실패 |
| 요청 시간 | `request_ts` | — |
| 응답 시간 | `received_ts` | — |

### 요청/응답 상세 모달 (DetailLogModal)

`보기` 버튼 클릭 시 `CommonDetailLogModal` 표시:

| 항목 | 값 |
|------|-----|
| 모달 제목 | "접속환경 요청 상세정보" 또는 "접속환경 응답 상세정보" |
| 파라미터 | `phry_id` (로그 ID) + `apiType` (`request_data` 또는 `response_data`) |

### API

| 동작 | API | 명세 |
|------|-----|------|
| 정책 로그 목록 | `GET /v1/policy/policyList` | — |
