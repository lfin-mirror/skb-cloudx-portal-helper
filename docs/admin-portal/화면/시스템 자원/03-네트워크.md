# 네트워크 (A1004, T1004)

경로: /system-resource/network/

네트워크 하위 4개 메뉴 통합 정리. 물리 네트워크, 서브넷, 라우터, QoS가 서로 어떻게 연결되는지 중심으로 설명.

## 연관 관계

- [OpenStack 네트워크 개념](../../openstack/시스템%20자원/04-neutron-네트워크.md)
- [테넌트 관리](../테넌트/01-테넌트.md) — 테넌트에 네트워크 할당

## 4개 메뉴의 관계

```
물리 네트워크 (A100401)          ← 인프라팀이 OpenStack에서 구성, SA 조회만
  │
  │  External Gateway (외부 연결)
  │
  ▼
라우터 (A100403, T100403)      ← TA가 생성, 가상↔물리 연결 다리
  │
  │  Internal Interface (내부 연결)
  │
  ▼
서브넷 (A100402, T100402)      ← TA가 생성, IP 범위 정의
  │
  │  Port (자동 생성)
  │
  ▼
VM (가상 PC)                    ← IP 할당받아 통신

네트워크 QoS (A100404, T100404) ← SA/TA가 생성, 네트워크에 적용하여 대역폭 제한
```

### 비유

```
물리 네트워크 = 건물의 인터넷 회선 (KT 광랜)
라우터       = 건물 공유기 (외부 회선 ↔ 내부 WiFi 연결)
서브넷       = WiFi 네트워크 (192.168.1.x 대역)
Port         = 노트북의 WiFi 연결 (IP 할당)
VM           = 노트북
QoS          = 대역폭 제한 (이 WiFi는 최대 100Mbps)
```

### SA vs TA 권한 요약

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| 보이는 네트워크 | — | 물리 (`P004P`) | 가상 (`P004V`) |
| 네트워크 CRUD | — | 조회만 | 생성/수정/삭제 가능 |
| 서브넷 | — | CRUD | CRUD |
| 라우터 | — | 조회만 (등록/삭제/저장 비노출, 외부NW 변경/서브넷 설정 disabled) | CRUD |
| QoS | — | CRUD | CRUD |
| | ↓ **usg_use_cd 값에 따라 편집 가능 여부 분기** | | |
| 네트워크 편집 | `usg_use_cd = 'P004P'` (물리) | 수정 불가 (`isEditable = false`) | 메뉴 자체 비노출 |
| | `usg_use_cd = 'P004V'` (가상) | 메뉴 자체 비노출 | CRUD 가능 (`isEditable = true`) |

### CIDR / DNS 개념

```
CIDR: 192.168.1.0/24
  → /24 = 256 IP (실제 사용 가능: 254개)
  → /16 = 65,536 IP
  → /8  = 16,777,216 IP

DNS: VM에서 도메인→IP 변환용
  → DNS 1: 주 DNS (예: 8.8.8.8)
  → DNS 2: 보조 DNS (DNS 1 장애 시 대체)
  → 서브넷에 설정하면 해당 서브넷의 모든 VM에 자동 적용
```

## 라우터 (A100403, T100403)

경로: /system-resource/network/router

컴포넌트: views/systemResource/network/Router.vue / RouterDetail.vue / RouterCreate.vue

가상 네트워크(서브넷)와 물리 네트워크를 연결하는 다리. 라우터가 없으면 VM은 외부와 통신 불가.

### 라우터의 두 가지 연결

```
┌─ 물리 네트워크 (외부) ─┐
│                        │
│  ┌─── 라우터 ───┐      │
│  │ External GW  │──────┘  ← 외부 연결 (물리 네트워크 1개만 지정)
│  │              │
│  │ Internal IF  │──── 서브넷 A (192.168.1.0/24)  ← 내부 연결 (여러 개 가능)
│  │ Internal IF  │──── 서브넷 B (192.168.2.0/24)
│  └──────────────┘
```

### SA vs TA 차이

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| **등록 버튼** | — | 비노출 | 노출 |
| **삭제 버튼** | — | 비노출 | 노출 |
| **저장 버튼** | — | 비노출 | 노출 |
| **외부 네트워크 변경** | — | disabled | 편집 가능 |
| **내부 인터페이스 설정/제거** | — | disabled | 편집 가능 |
| **상세 폼** | — | 조회만 | 수정 가능 |

### 테이블 컬럼

| 화면 라벨 | 필드 |
|----------|------|
| 라우터명 (`list.systemResource.routerName`) | `rtr_nm` |
| 외부 네트워크 (`list.systemResource.externalNetwork`) | `nw_nm` |
| 내부 인터페이스 수 (`list.systemResource.internalInterfaceCnt`) | `sbn_cnt` |
| 생성자 (`list.common.creator`) | `reg_conn_id` |
| 생성일시 (`list.common.createDate`) | `reg_ts` |

### 행 클릭 → 라우터 상세 (RouterDetail)

| 필드 | 설명 | SA | TA |
|------|------|----|----|
| 라우터명 (`rtr_nm`) | | disabled | disabled |
| 외부 네트워크 (`nw_nm`) | 변경 버튼 → CommonSingleModal (물리 네트워크 `P004P` + `in_tnt_id` 필터) | disabled | 편집 가능 |
| 내부 인터페이스 테이블 | 서브넷명 + CIDR | 조회만 | 설정/제거 가능 |

### 등록 폼 (RouterCreate, TA 전용)

|------|------|------|
| 라우터명 (`rtr_nm`) | 필수 | |


## 네트워크 QoS (A100404, T100404)

경로: /system-resource/network/qos

컴포넌트: views/systemResource/network/NetQos.vue / NetQosDetail.vue (SA/TA 동일 컴포넌트)

네트워크 대역폭을 제한하는 정책. 네트워크에 QoS 정책을 적용하면 해당 네트워크의 모든 Port(= 모든 VM)에 대역폭 제한 적용. SA/TA 차이 없음.

### 테이블 컬럼

| 화면 라벨 | 필드 | 비고 |
|----------|------|------|
| 정책명 (`list.systemResource.nwQosName`) | `nw_qos_plcy_nm` | |
| 규칙 | `rules` | Ingress/Egress별 최대속도/버스트를 ul/li 렌더링 |
| 생성자 (`list.common.creator`) | `reg_conn_id` | |
| 생성일시 (`list.common.createDate`) | `reg_ts` | |

### 행 클릭 → QoS 상세 (NetQosDetail)

| 필드 | 설명 | 편집 |
|------|------|------|
| 정책명 (`nw_qos_plcy_nm`) | | disabled (수정 시) |
| Ingress 체크박스 | 수신 규칙 on/off | 편집 가능 |
| Egress 체크박스 | 송신 규칙 on/off | 편집 가능 |

체크 시 해당 방향의 규칙 입력 필드 노출:

| 필드 | 설명 |
|------|------|
| 최대 속도 (`max_kbps`) | kbps 단위 |
| 버스트 (`max_burst_kbps`) | kbps 단위 |

### 등록 폼 (NetQosDetail, 신규)

## 전체 연결 흐름 — VM이 외부 인터넷에 접속하기까지

```
1. [인프라팀] 물리 네트워크 구성 (OpenStack)
     → VLAN 기반, 물리 스위치와 연결
     → /initialized 마법사에서 로컬 DB 동기화

2. [TA] 가상 네트워크 생성
     → P004V 유형, 테넌트 전용

3. [TA] 서브넷 생성
     → CIDR: 192.168.1.0/24, Gateway: 192.168.1.1
     → DNS, DHCP 설정

4. [SA/TA] QoS 정책 생성 (선택)
     → 대역폭 제한, 네트워크에 적용

5. [TA] 라우터 생성
     → External Gateway: 물리 네트워크 지정
     → Internal Interface: 서브넷 추가

6. [TA] 가상 PC 할당 (VM 생성)
     → VM이 서브넷에 연결 → Port 자동 생성 → IP 할당

7. VM → Port → 서브넷 → 라우터 → 물리 네트워크 → 외부 인터넷
```
