# 스토리지 (A1005, T1005) / 네트워크 스토리지 (A1007, T1007)

저장소 관련 메뉴 통합. 블록 스토리지(Cinder 볼륨)와 네트워크 스토리지(NAS 공유 폴더).

## 연관 관계

- [OpenStack Cinder 개념](../../openstack/시스템%20자원/05-cinder-블록-스토리지.md)
- [가상 디스크 메뉴](../가상%20디스크/01-가상%20디스크.md) — TA가 Volume을 VM에 연결/분리

## 블록 스토리지 3개 메뉴의 관계

```
Backend (NetApp 등 실제 스토리지 하드웨어)
  │
  └── 볼륨 타입 (A100501, T100501) ← 프로필 정의
        │
        ├── 디스크 QoS (A100502) ← 성능 제한
        │
        └── 볼륨 자원 (A100503, T100503) ← 현황 조회
              │
              └── Volume (디스크) → VM에 연결
```

### SA vs TA 권한 요약

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| **볼륨 타입** | 조회 범위 | 전체 조회 | 자기 테넌트 것만 |
| | 할당 테넌트 컬럼 | 표시 | 미표시 |
| | 행 클릭 상세 | 사용 타입/QoS 정책/설명 수정 가능 | 상세 패널 미구현 (Block.vue의 `get()` 즉시 return) |
| **디스크 QoS** | — | CRUD (SA 전용) | 메뉴 없음 |
| **볼륨 자원** | — | 전체 조회 | 자기 테넌트 볼륨만 조회 |
| **네트워크 스토리지** | — | 전체 테넌트 집계 | 사용자 그룹 단위 상세 |

## 볼륨 타입 (A100501, T100501)

SA 경로: /system-resource/storage/volume-Type — 컴포넌트: VolumeType.vue

TA 경로: /system-resource/storage/block-storage — 컴포넌트: Block.vue

"이 디스크를 어떤 스토리지 장비에서, 어떤 방식으로 제공할 것인가"를 정의하는 프로필. SA 등록 버튼 비활성화(CLOUDPCQA-1778).

### SA vs TA 테이블 컬럼 차이

| 화면 라벨 | 필드 | SA | TA |
|----------|------|----|----|
| 볼륨 타입 명 (`list.systemResource.volumeTypeName`) | `volm_typ_nm` | O | O |
| 백엔드 명 (`list.systemResource.backendName`) | `volm_be_nm` | O | O |
| 사용 유형 (`list.systemResource.useType`) | `volm_usg_typ_nm` | O | O |
| 사용량/총량 | `usg_capa`/`tot_capa` | O | O |
| QoS 정책 (`list.systemResource.diskQosName`) | `volm_qos_plcy_nm` | O | O |
| 할당 테넌트 (`list.systemResource.allocateTenant`) | `assignedTenant` | O (SA 전용, `tnt_list[].tnt_nm` ul/li 렌더링) | X |
| 생성자 (`list.common.creator`) | `reg_conn_id` | O | O |
| 생성일시 (`list.common.createDate`) | `reg_ts` | O | O |

### 행 클릭 → SA 볼륨 타입 상세 (VolumeTypeDetail)

| 필드 | 설명 | 편집 |
|------|------|------|
| 볼륨 타입 명 (`volm_typ_nm`) | | disabled |
| 백엔드 명 (`volm_be_nm`) | | disabled |
| 사용 타입 (`volm_usg_typ`) | 셀렉트 (코드 U034: 겸용/가상PC용/개인디스크용) | 편집 가능 |
| 사용량/총량 | `usg_capa`/`tot_capa` | disabled |
| QoS 정책 (`volm_qos_plcy_id`) | 셀렉트 — 미연결 QoS(`volm_typ_id === null`) + 현재 연결 QoS만 표시 | 편집 가능 |
| 설명 (`desc`) | textarea 150자 | 편집 가능 |

QoS 정책 셀렉트가 사실상 **associate 방법** — VolumeTypeDetail에서 QoS를 선택하여 저장하면 볼륨 타입에 QoS가 연결됨.

TA(Block.vue)는 행 클릭 시 `get()` 메서드가 즉시 `return` → **상세 패널 미구현**.

## 디스크 QoS (A100502) — SA 전용

경로: /system-resource/storage/disk-qos

컴포넌트: views/systemResource/storage/DiskQos.vue / DiskQosDetail.vue

SA 전용 (TA 메뉴에 없음). 디스크 I/O 성능 제한 정책 CRUD.

### 규칙 단위

| 단위 | 코드 | 설명 |
|------|------|------|
| IOPS | `V018IOP` | 초당 I/O 작업 수 |
| Bandwidth (MB/s) | `V018BAN` | 초당 전송량 |

### 테이블 컬럼

| 화면 라벨 | 필드 | 비고 |
|----------|------|------|
| 정책명 | `volm_qos_plcy_nm` | |
| 볼륨 타입 | `volm_typ_nm` | 연결된 볼륨 타입 |
| 규칙 단위 | `volm_qos_rule_unit_cd_nm` | IOPS / Bandwidth |
| QoS 규칙 | `rules` | Read/Write/ReadMax/WriteMax를 ul/li 렌더링 |
| 생성자 | `reg_conn_id` | |
| 생성일시 | `reg_ts` | |

### 행 클릭 → 디스크 QoS 상세 (DiskQosDetail)

| 필드 | 설명 | 편집 |
|------|------|------|
| 정책명 (`volm_qos_plcy_nm`) | | 등록 시 필수, 수정 시 disabled |
| 규칙 단위 (`volm_qos_rule_unit_cd`) | 셀렉트 (IOPS/Bandwidth) | 편집 가능 |
| QoS Rule 테이블 | 읽기/쓰기/읽기 최대/쓰기 최대 | InputNumber (0~999999999) |

IOPS/Bandwidth 전환 시 별도 값 유지 (`iopsQosRule`/`bandQosRule`).

### 삭제 조건

`form.volm_typ_id === null`일 때만 삭제 버튼 노출 — 볼륨 타입에 연결된 QoS는 삭제 불가.

### 볼륨 타입과의 연결

디스크 QoS 화면 자체에는 연결 기능이 없음. **VolumeTypeDetail에서 QoS 정책 셀렉트로 연결**.

### 버튼 동작

| 버튼 | 클릭 시 동작 |
|------|------------|
| **저장** (등록) | `POST /v1/resource/storage/qos` |
| **저장** (수정) | `PUT /v1/resource/storage/qos/{id}` |
| **삭제** | `volm_typ_id === null`일 때만 노출. 확인 팝업 → `DELETE /v1/resource/storage/qos/{id}` |

### API

| 동작 | 경로 | 명세 |
|------|------|------|
| 목록 | `GET /v1/resource/storage/qos` | [명세](../../api/resource/storage/01-nas.md) |
| 상세 | `GET /v1/resource/storage/qos/{id}` | [명세](../../api/resource/storage/01-nas.md) |
| 생성 | `POST /v1/resource/storage/qos` | [명세](../../api/resource/storage/01-nas.md) |
| 수정 | `PUT /v1/resource/storage/qos/{id}` | [명세](../../api/resource/storage/01-nas.md) |
| 삭제 | `DELETE /v1/resource/storage/qos/{id}` | [명세](../../api/resource/storage/01-nas.md) |

## 볼륨 자원 (A100503, T100503)

경로: /system-resource/storage/volume-resource

컴포넌트: views/systemResource/storage/VolumeResource.vue / VolumeResourceDetail.vue

실제 생성된 디스크(Volume)들의 현황 조회. 읽기 전용. SA는 전체, TA는 자기 테넌트만.

### 화면 구성

```
┌─ 탭 (ALL / 사용유형별) ──────────────────────┐
│  탭 클릭 → volm_usg_typ 파라미터로 필터        │
├─ 카드 UI (볼륨 타입별 선택) ──────────────────┤
│  카드 내용: {allo_vm_cnt}(가상 PC) / {allo_dsk_cnt}(볼륨) │
│  카드 색상: U034BOT=#2abc9b, U034VPC=#367aab, 그 외=#565656 │
│  카드 클릭 → volm_typ_id 파라미터로 필터        │
├─ 테이블 (선택한 타입의 디스크 목록) ────────────┤
└──────────────────────────────────────────────┘
```

### 테이블 컬럼

## 네트워크 QoS vs 디스크 QoS

| 항목 | 네트워크 QoS (A100404) | 디스크 QoS (A100502) |
|------|----------------------|---------------------|
| 대상 | 네트워크 트래픽 | 디스크 I/O |
| 단위 | kbps (대역폭) | IOPS 또는 MB/s |
| 적용 대상 | 네트워크 → 모든 Port | 볼륨 타입 → 해당 타입 모든 디스크 |
| SA/TA | 공통 | SA 전용 |

## 네트워크 스토리지 (A1007, T1007)

경로: /system-resource/nws

컴포넌트: views/systemResource/storage/Nas.vue

공유 폴더(NAS) 사용량 조회. 행 클릭 없음 (상세 화면 없음). SA/TA 분기 기준: `grp_typ_cd === 'U001TNT'`이면 TA.

### SA vs TA 차이

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| **범위** | — | 전체 테넌트 집계 | 자기 테넌트 사용자 그룹 단위 |
| **API** | — | `POST /v1/legacy/cloud/ad/allSharedfolders` | `POST /v1/legacy/cloud/ad/sharedfolder/directories` |
| **검색 조건** | — | 테넌트명 (`tnt_nm`) | 스토리지명 (`dept_cd`) + 사용자 그룹 (`usr_grp_nm`) |

### SA 테이블 컬럼

| 화면 라벨 | 필드 |
|----------|------|
| 테넌트 | `tnt_nm` |
| 사용량 | `usedSize` |
| 총량 | `totalSize` |

### TA 테이블 컬럼

| 화면 라벨 | 필드 |
|----------|------|
| 사용자 그룹 | `usr_grp_nm` |
| 스토리지명 | `dept_cd` |
| 사용량 | `used_size` |
| 총량 | `total_size` |

SA는 camelCase(`usedSize`), TA는 snake_case(`used_size`) — 필드명 형식이 다름.
