## 마이그레이션 (A1006, T1006)

경로: /system-resource/migration

컴포넌트: views/systemResource/Migration.vue

VM 마이그레이션 작업 목록 및 상태 모니터링. 8초 주기 자동 갱신. 내부적으로 Live Migration 사용.

### SA vs TA 차이

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| **범위** | — | 전체 VM | 자기 테넌트 VM만 |
| **등록** | — | 가능 | 가능 |
| **초기화** | — | 가능 | 가능 |

### 검색 조건

| 검색 항목 | 필드 |
|----------|------|
| 상태 | 셀렉트박스 (코드 J008) |

### 마이그레이션 상태

| 상태 | 색상 | 설명 |
|------|------|------|
| 작업 중 | 파랑 | 데이터 복사 중 |
| 실패 | 빨강 | 에러 발생 |
| 완료 | 초록 | 성공적으로 이동 |

### 테이블 컬럼

| 화면 라벨 | 필드 | 비고 |
|----------|------|------|
| 상태 | `job_sts_cd` | 색상 표시 (파랑/빨강/초록) |
| 원본 VM | `src_vm_nm` | |
| 원본 호스트 | `src_host_nm` | |
| 대상 호스트 | `tgt_host_nm` | |
| 생성자 | `reg_conn_id` | |
| 시작 시간 | `job_stt_tm` | |
| 완료 시간 | `job_done_tm` | |
| VM 상태 | `src_vm_state` | |
| 초기화 | — | 하드코딩 "초기화" 버튼 |

### 초기화 버튼

노출 조건 (3가지 모두 충족):
- `job_sts_cd === 'J008F'` (실패)
- `src_vm_state === 'active'` (VM 활성)
- `src_rstr_sts_cd === 'U022MGF'` (마이그레이션 실패 상태)

클릭 → 확인 팝업 "초기화 하시겠습니까?" → `POST /v1/resource/vpcs/migration/{mig_id}/initial`

### 행 클릭 → 마이그레이션 상세 (MigrationDetail)

5초 폴링으로 상태 갱신. 모든 필드 readonly.

| 필드 | 설명 |
|------|------|
| 원본 VM (`src_vm_nm`) | |
| 원본 호스트 (`src_host_nm`) | |
| 대상 호스트 (`tgt_host_nm`) | |
| 시작 시간 (`job_stt_tm`) | |
| 완료 시간 (`job_done_tm`) | |

### 등록 (MigrationCreate)

| 단계 | 필드 | 설명 |
|------|------|------|
| 1 | 원본 호스트 선택 | 모달에서 호스트 선택 |
| 2 | 대상 호스트 선택 | 모달에서 호스트 선택 |
| 3 | 가상 PC 선택 | 원본 호스트의 VM 목록에서 선택 |

등록 → `POST /v1/resource/vpcs/migration/multi`

### API

| 동작 | 경로 | 명세 |
|------|------|------|
| 목록 조회 | `GET /v1/resource/vpcs/migration` | [명세](../../api/resource/migration/01-migration.md) |
| 상세 조회 | `GET /v1/resource/vpcs/migration/{mig_id}` | [명세](../../api/resource/migration/01-migration.md) |
| 등록 | `POST /v1/resource/vpcs/migration/multi` | [명세](../../api/resource/migration/01-migration.md) |
| 초기화 | `POST /v1/resource/vpcs/migration/{mig_id}/initial` | [명세](../../api/resource/migration/01-migration.md) |

## HA 실행 (A101101, T100801)

경로: /system-resource/evacuate/evacuate

컴포넌트: views/systemResource/EvacuateList.vue

호스트 장애 시 VM 대피(Evacuate) 실행 및 상태 모니터링.

### SA vs TA 차이

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| **범위** | — | 전체 호스트 | 자기 테넌트 호스트만 |
| **HA 설정 버튼** | — | 표시 (`v-if="isSuperAdmin"`) | 비노출 |
| **수동 실행** | — | 가능 | 가능 |

### 상태 범례

| 상태 | 코드 | 색상 |
|------|------|------|
| 정상 | `P003NOR` | 초록 |
| 다운 | `P003DWN` | 빨강 |

### 테이블 컬럼

| 화면 라벨 | 필드 | 비고 |
|----------|------|------|
| 상태 | `host_sts_cd` | StatusPane (P003) |
| 호스트명 | `host_nm` | |
| 위험 발생 시간 | `dng_sts_tm` | |
| 실행 | — | 아래 3가지 상태로 동적 표시 |

### 실행 컬럼 동적 표시

| 조건 | 표시 |
|------|------|
| `execBtn === 'Y'` | **실행** 버튼 |
| `evac_sts_cd === 'J010ING'` | "진행 중" 텍스트 |
| 그 외 | "처리 완료" 텍스트 |

### 버튼 동작

| 버튼 | 노출 조건 | 클릭 시 동작 |
|------|----------|------------|
| **수동 실행** | `execBtn === 'Y'` | 확인 팝업 없이 바로 `POST /v1/resource/evacuate/{host_id}` (Vuex `manualExecution`) |
| **HA 설정** | SA 전용 (`v-if="isSuperAdmin"`) | EvacuationSettingFor 모달 열기 |

### HA 설정 모달 (SA 전용)

| 필드 | 설명 |
|------|------|
| 자동 실행 (`evacuate.enable`) | on/off |
| 유예 시간 (`evacuate.gracePeriod`) | 분 단위 |

| 동작 | API |
|------|-----|
| HA 설정 조회 | `GET /v1/system/settings?category=evacuate` |
| HA 설정 저장 | `PUT /v1/system/settings` |

### API

| 동작 | API | 명세 |
|------|-----|------|
| 목록 조회 | `GET /v1/resource/evacuate` | (미작성) |
| 수동 실행 | `POST /v1/resource/evacuate/{host_id}` | (미작성) |
| HA 설정 조회 (SA) | `GET /v1/system/settings?category=evacuate` | (미작성) |
| HA 설정 저장 (SA) | `PUT /v1/system/settings` | (미작성) |
