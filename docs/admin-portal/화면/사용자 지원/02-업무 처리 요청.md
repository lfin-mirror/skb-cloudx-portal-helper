# 업무 처리 요청 (T0302)

- 경로: `/user-support/job-work-request`
- 컴포넌트: `JobWorkRequest.vue`
- 상세: `JobWorkRequestDetail.vue`

user-portal에서 사용자가 신청한 모든 요청을 TA가 조회·처리하는 메뉴. 장애 처리, 자원 할당, 반납, 기간 연장, 보안 정책 변경 등 **모든 유형의 요청이 이 한 곳에 집결**.

## UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조. 행 클릭 시 좌측에 상세 폼 표시.

### 필터

| 필터 | 필드 | 설명 |
|------|------|------|
| 진행 상태 | `pgrs_sts_cd` | 신청/승인/반려/취소 |
| 요청 유형 | `usr_req_div_cd` | J003 코드 그룹 (아래 참고) |

### 상태 표시 (StatusPane)

| 색상 | 코드 | 의미 |
|------|------|------|
| yellow | `J001S` | 신청 |
| green | `J001A` | 승인 |
| red | `J001R` | 반려 |
| gray | `J001C` | 취소 |

### 테이블 컬럼

| 컬럼 | 필드 | 설명 |
|------|------|------|
| 상태 | `pgrs_sts_cd` | 색상 아이콘 |
| 요청 유형 | `usr_req_div_cd_nm` | 코드명 (장애 처리, 자원 할당 등) |
| 요청자 ID | `req_acct_conn_id` | — |
| 요청일시 | `demd_tm` | — |
| 처리자 ID | `had_acct_conn_id` | — |
| 처리일시 | `had_tm` | — |

## 요청 유형 코드 (`usr_req_div_cd`, J003 그룹)

| 코드 | 의미 | user-portal 메뉴 |
|------|------|-----------------|
| `J003ACC` | 계정 연장 | — |
| `J003USE` | 자원 할당 | vPcInfo > VPcReq |
| `J003ADD` | 자원 증설 | — |
| `J003RET` | 자원 회수 | vPcInfo > VPcReturn |
| `J003ERR` | 장애 처리 | support > DisReqReg |
| `J003AUT` | 정책 변경 | — |
| `J003PET` | 가상 PC 기간 연장 | VpcInfo > VPcPeriodExt |
| `J003DAR` | 단말 접속 정보 등록 | vPcInfo > DeviceAccReq |

## 상세 화면 — 유형별 분기 (JobWorkRequestDetail)

요청 유형(`usr_req_div_cd`)에 따라 상세 폼 내용이 달라짐:

### J003USE (자원 할당)

가상 PC 리소스 선택 UI 표시. VM 할당 처리.

- `GET /v1/resource/vpcs/resources` — 할당 가능한 VM 목록
- `GET /v1/resource/vpcs/resources/{id}` — VM 상세

### J003ADD (자원 증설)

CPU, 메모리, 디스크 증설 요청 내용 표시. TA가 승인하면 VM 스펙 변경.

### J003AUT (보안 정책 변경)

보안 정책 관련 폼 표시:

- `GET /v1/operation/cert/secu/adopter/info/{id}` — 현재 보안 정책 조회
- `POST /v1/operation/cert/secu/adopter/info` — 변경된 보안 정책 저장
- `GET /v1/operation/policy/excn/excngrp/{id}` — 예외 네트워크 그룹

### J003PET (기간 연장) / J003ACC (계정 변경)

기간/계정 정보 표시. 단순 승인/반려.

### J003DAR (단말 접속 정보 등록)

단말 식별 정보 표시. 승인 시 해당 단말에서 VM 접속 허용.

### J003ERR (장애 처리)

장애 내용 표시. TA가 처리 후 상태 변경.

### 공통 동작

- `J001S`(신청) 상태에서만 `요청 취소` 버튼 노출
- TA가 상세에서 승인(`J001A`) 또는 반려(`J001R`) 처리

## user-portal → admin-portal 흐름

```
[user-portal]                              [admin-portal]
사용자가 신청                               TA가 처리
     │                                          │
     │ POST /v1/user/work/request               │
     │ (usr_req_div_cd: J003ERR 등)             │
     ▼                                          ▼
  신청 완료                               업무 처리 요청 (T0302)
  DisReqList에서 본인 이력 확인              목록에서 J001S(신청) 상태 확인
     │                                     행 클릭 → 상세 폼
     │                                     유형별 처리 후 승인/반려
     ▼                                          │
  상태 변경 확인                                  │
  (J001S → J001A 또는 J001R)              ◄──────┘
```
