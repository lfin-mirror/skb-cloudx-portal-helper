# 가상 PC 원격 제어 (T0301)

- 경로: `/user-support/virtual-pc-remote-control`
- 컴포넌트: `VirtualPcRemoteControl.vue`
- 상세: `VirtualPcRemoteControlDetail.vue` (플레이스홀더)

TA가 사용자의 VM에 원격으로 접속하는 기능. 장애 대응·기술 지원 목적.

## UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조.

### 필터

| 필터 | 필드 | 설명 |
|------|------|------|
| 접속 상태 | `usr_vm_conn_sts_cd` | 접속 중 / 접속 종료 |
| 전원 상태 | `vm_power_sts_cd` | 전원 켜짐 / 꺼짐 / 장애 등 |

### 상태 표시 (StatusPane)

전원 상태와 접속 상태를 색상으로 구분:

| 색상 | 전원 상태 |
|------|----------|
| blue | 전원 켜짐 |
| purple | 전원 꺼짐 |
| black | 접속 중 |
| gray | 접속 종료 |
| yellow | 복구 중 |
| red | 장애 |

### 원격 접속 버튼

테이블 행에 `원격 접속` 버튼. 클릭 시 `VirtualPcRemote` 모달 실행:

```
1. POST /sm/v1/cloudpc/vpcs/{vmId}/remote/request → 원격 접속 요청 전송
2. 5초 간격 폴링: GET /sm/v1/cloudpc/vpcs/{vmId}/remote/request/{requestKey}
3. 상태 분기:
   - APPROVED → skbvdi://cloudpc?...&REMOTE=1 로 VDI 클라이언트 실행
   - PROGRESS → 대기 계속
   - REJECTED → "접속 거부" 안내
```

사용자 측에서 승인해야 관리자가 원격 접속 가능.

## API

| 동작 | API | 명세 |
|------|-----|------|
| VM 목록 | `GET /v1/resource/vpcs/resources` | [명세](../../api/resource/vpcs/01-vpcs.md) |
| 원격 접속 요청 | `POST /sm/v1/cloudpc/vpcs/{vmId}/remote/request` | (미작성) |
| 원격 접속 상태 | `GET /sm/v1/cloudpc/vpcs/{vmId}/remote/request/{requestKey}` | (미작성) |

## user-portal과의 관계

user-portal 측에서 사용자에게 원격 접속 승인/거부 알림이 표시됨 (추측). 관리자가 요청을 보내면 사용자가 승인해야 `APPROVED` 상태로 전환.
