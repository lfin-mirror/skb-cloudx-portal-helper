# 시스템 설정 (A1204 / A1205 / A1206 / A1209)

SA 전용 (TA 메뉴에 없음). 플랫폼 전반의 라이선스, 버전, 코드, 환경 설정을 관리하는 4개 화면.

## 4개 화면의 관계

```
CloudX 플랫폼
  ├── 시스템 라이선스 (A1205) — 가상 PC 허용 수량
  ├── S/W 버전 정보 (A1206) — 플랫폼 컴포넌트 버전
  ├── 코드 관리 (A1204) — 공통코드 원본
  └── 환경 설정 (A1209) — 전역 ON/OFF 설정
```

---

## 시스템 라이선스 (A1205)

경로: /admin-setting/system-license

컴포넌트: views/adminSetting/SystemLicense.vue

플랫폼 라이선스 정보 조회. 읽기 전용.

| 항목 | 설명 |
|------|------|
| `version` | 라이선스 버전 |
| `license_valid_period` | 유효 기간 |
| `quantity_virtual_pc` | 허용 가상 PC 수량 |
| `contact` | 연락처 |

API: `GET /v1/operation/system/license` — [명세](../../api/operation/system/01-license.md)

---

## S/W 버전 정보 (A1206)

경로: /admin-setting/sw-version

컴포넌트: views/adminSetting/SwVersion.vue

플랫폼 소프트웨어 버전 조회. 읽기 전용. 운영팀이 각 마이크로서비스의 배포 버전 확인용.

| 컬럼 | 필드 | 예시 |
|------|------|------|
| 소프트웨어명 | `sw_nm` | `app-ms-resource`, `plat-ms-vid4o` 등 |
| 서브 모듈명 | `sub_sw_nm` | 내부 모듈 |
| 버전 | `ver` | `2.3.1` |

API: `GET /v1/resource/platform/software` — [명세](../../api/resource/templates/01-templates.md)

---

## 코드 관리 (A1204)

경로: /admin-setting/code

컴포넌트: views/adminSetting/Code.vue

공통 코드 분류 조회. 읽기 전용. admin-portal 전체에서 사용하는 공통코드의 원본.

`plugins/codes.js`의 `$codes.get(key)`가 이 코드를 조회하여 `sessionStorage.codes`에 캐싱.

```
코드 예시:
  A003 → 1:1문의 유형
  P004 → 네트워크 유형 (P004P: 물리, P004V: 가상)
  U001 → 관리자 역할 (U001SUP: SA, U001TNT: TA)
  U013 → 계정 상태 (U013IN: 초기화, U013NR: 정상, U013SL: 휴면)
```

API: `GET /v1/system/commons/codes/groups` — [명세](../../api/system/commons/01-codes.md)

## 환경 설정 (A1209)

경로: /admin-setting/environment-setting

컴포넌트: views/adminSetting/EnvironmentSetting.vue

플랫폼 전역 설정 관리.

| 필드 | 설명 | 상태 |
|------|------|------|
| `keylogging_protect_yn` | 키로깅 방지 사용 여부 | 활성 |
| `unsupported_os_allow_yn` | 미지원 OS 접속 허용 여부 | 비활성 (disabled) |

키로깅 방지 = `true` 설정 시 admin-portal 전체 로그인 흐름에 영향: 매 네비게이션마다 에이전트 연결 확인 → 미설치/구버전 시 모달 표시.

| 동작 | 경로 |
|------|------|
| 조회 | `GET /v1/system/settings?category=admin` |
| 저장 | `PUT /v1/system/settings` |
