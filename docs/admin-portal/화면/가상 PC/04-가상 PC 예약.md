# 가상 PC 예약 (T0704)

경로: /virtual-pc/virtual-pc-reservate

컴포넌트: views/virtualPc/VirtualPcReservate.vue

TA 전용. `isSuperAdmin` 분기 없음.

VM 할당 예약 상태 모니터링. 할당 준비/실패 상태를 추적하고, 실패 시 재시도 또는 자원 회수.

## 연관 관계

```
[가상 PC 그룹] (T0701) 풀에서 "할당" 버튼 클릭
     │
     │  VM 할당 요청 (비동기)
     ▼
[가상 PC 예약] (T0704) 할당 진행 상태 모니터링
     │
     ├── 준비 중 (U017ALR/U017RAR) → 8초 간격 자동 갱신
     ├── 성공 → [가상 PC] (T0702) 목록에 표시
     └── 실패 (U017ALF/U017RAF) → 재시도 또는 자원 회수
```

## UI 레이아웃

상단 Pool 선택 탭 (전용/공용) + 하단 예약 상태 리스트. 8초 간격 자동 갱신.

### 전용/공용 PC 탭

| 탭 | `tnt_mtd_cd` |
|----|-------------|
| 전체 | `''` (빈값) |
| 전용 PC | `V001DED` |
| 공용 PC | `V001POO` |

### 상태 필터

목록은 아래 4개 상태만 표시 (성공한 건은 안 보임):

| 상태 코드 | 의미 |
|----------|------|
| `U017ALR` | 할당 준비 |
| `U017ALF` | 할당 실패 |
| `U017RAR` | 재할당 준비 |
| `U017RAF` | 재할당 실패 |

### 테이블 컬럼

| 컬럼 | 필드 | 설명 |
|------|------|------|
| PC 타입 | `tnt_mtd_cd_nm` | 전용/공용 |
| Pool명 | `vm_pool_nm` | — |
| 사용자 그룹 | `usr_grp_nm` | — |
| 사용자명 | `acct_nm` | — |
| 고정 IP 할당 여부 | `vm_allo_typ_static_ip_yn` | — |
| 할당 상태 | `vm_allo_sts_cd` | — |

### 조건부 버튼

| 버튼 | 표시 조건 | 클릭 시 동작 |
|------|----------|------------|
| 실패 사유 | 할당 실패 또는 재할당 실패 시 | `ErrorDialog` 모달 표시 — `allo_fail_msg` 필드의 실패 메시지 |
| 재시도 | 고정 IP가 아니고 + 할당 실패/재할당 실패 시 | 확인 팝업(`isRetryAssign`) → `postVirtualPcInit` (params: `user_workset_yn: 'Y'`) |
| 자원 회수 | 할당 실패/재할당 실패 시 | 확인 팝업(`isRecallTheResource`) → `deleteVmTerminate` (params: `user_workset_yn: 'Y'`) |

"재시도" 버튼은 고정 IP 할당(`vm_allo_typ_static_ip_yn = 'Y'`)이면 **비노출** — 고정 IP는 자동 재배정이 불가하므로.
