# SW 제어 정책 (A050604/T050604, A050605/T050605, A050606/T050606, A050607/T050607)

상세 정책(A0506) 하위의 소프트웨어/URL 관련 4개 메뉴를 통합 정리.

## 4개 메뉴 개요

| 메뉴 | 메뉴ID | 목적 |
|------|--------|------|
| 블랙리스트 | A050604 | VM에서 실행 차단할 SW 목록 |
| 화이트리스트 | A050605 | VM에서 실행 허용할 SW 목록 |
| URL 리다이렉션 금지 | A050606 | VM에서 리다이렉션 차단할 URL 목록 |
| 미라이선스 정책 | A050607 | 라이선스 없는 VM의 보안 정책 |

### 관계

```
블랙리스트 ←→ 화이트리스트     블랙 = "이것만 차단", 화이트 = "이것만 허용"
    │              │           (동일 API 경로, 파라미터로 구분)
    │              │
    └──── 가상 PC 보안에 적용 ────┘

URL 리다이렉션 금지             VM 브라우저에서 특정 URL 리다이렉션 차단

미라이선스 정책                  라이선스 미인증 VM에 적용할 보안 수준
```

---

## 블랙리스트 (A050604)

- 경로: `/policy/policy-manage/black-list`
- 컴포넌트: `BlacklistUpload.vue`

VM에서 **실행을 차단**할 소프트웨어 목록 관리.

### UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조. 상단에 SW명 검색 + 버전 표시.

### 테이블 컬럼

| 컬럼 | 필드 |
|------|------|
| SW명 | `name` |
| 제조사 | `company` |
| 원본명 | `origname` |
| 발행자 | `issuer` |

### 등록 방식

1. **개별 등록**: 행 클릭 → 좌측 상세 폼에서 직접 입력
2. **JSON 파일 업로드**: `button.policy.uploadJson` 버튼 → JSON 파일로 일괄 등록
3. **JSON 다운로드**: 현재 목록을 JSON 파일로 내보내기

### API

| 동작 | API | 명세 |
|------|-----|------|
| 목록 | `GET /v1/operation/policys/swblst` | [명세](../../api/operation/blacklist/01-blacklist.md) |
| 상세 | `GET /v1/operation/policys/swblst/{id}` | [명세](../../api/operation/blacklist/01-blacklist.md) |
| 다운로드 | `GET /v1/operation/policys/swblst/{id}/downLoad` | [명세](../../api/operation/blacklist/01-blacklist.md) |

## 화이트리스트 (A050605)

- 경로: `/policy/policy-manage/white-list`
- 컴포넌트: `WhitelistUpload.vue`

VM에서 **실행을 허용**할 소프트웨어 목록 관리. 블랙리스트와 동일한 API, 파라미터로 구분.

### API

| 동작 | API | 비고 | 명세 |
|------|-----|------|------|
| 목록 | `GET /v1/operation/policys/swblst` | `params`로 화이트리스트 구분 | [명세](../../api/operation/blacklist/01-blacklist.md) |
| 상세 | `GET /v1/operation/policys/swblst/{id}` | 동일 | [명세](../../api/operation/blacklist/01-blacklist.md) |
| 다운로드 | `GET /v1/operation/policys/swblst/{id}/downLoad` | 동일 | [명세](../../api/operation/blacklist/01-blacklist.md) |

블랙리스트와 화이트리스트는 **같은 API 엔드포인트**를 공유하고 요청 파라미터(`params`)로 타입을 구분.

### UI/테이블

블랙리스트와 동일한 구조 (SW명, 제조사, 원본명, 발행자). JSON 업로드/다운로드도 동일.

## URL 리다이렉션 금지 (A050606)

- 경로: `/policy/policy-manage/url-Redirection-disallow`
- 컴포넌트: `UrlRedirectionDisallow.vue`

VM 내 브라우저에서 특정 URL로의 리다이렉션을 차단하는 정책.

### UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조. 개별 등록과 일괄 등록(파일 업로드) 지원.

### 등록 방식

1. **개별 등록**: `등록` 버튼 → 좌측 상세 폼
2. **일괄 등록**: `일괄 등록` 버튼 → `UrlRedirectionUpload` 컴포넌트로 파일 업로드

### API

| 동작 | API | 명세 |
|------|-----|------|
| 목록 | `GET /v1/operation/policy/url/rdrt/disallow/policy` | [명세](../../api/operation/url-redirection/01-url-redirection.md) |
| 상세 | `GET /v1/operation/policy/url/rdrt/disallow/policy/{id}` | [명세](../../api/operation/url-redirection/01-url-redirection.md) |
| 생성 | `POST /v1/operation/policy/url/rdrt/disallow/policy` | [명세](../../api/operation/url-redirection/01-url-redirection.md) |
| 수정 | `PUT /v1/operation/policy/url/rdrt/disallow/policy/{id}` | [명세](../../api/operation/url-redirection/01-url-redirection.md) |
| 삭제 | `DELETE /v1/operation/policy/url/rdrt/disallow/policy/{id}` | [명세](../../api/operation/url-redirection/01-url-redirection.md) |

## 미라이선스 정책 (A050607)

- 경로: `/policy/policy-manage/unlicensed-policy`
- 컴포넌트: `UnlicensedPolicyManage.vue`

라이선스가 인증되지 않은 VM에 적용할 보안 정책. VM 클라이언트가 라이선스 서버와 통신 실패 시 이 정책에 따라 동작.

### API

| 동작 | API | 명세 |
|------|-----|------|
| 목록 | `GET /v1/operation/cert/secu/noauth` | [명세](../../api/operation/cert/01-cert.md) |
| 상세 | mixin의 `api.policyDetail` 경로 | [명세](../../api/operation/cert/01-cert.md) |
| 생성 | mixin의 `api.policyRegister` 경로 | [명세](../../api/operation/cert/01-cert.md) |
| 수정 | mixin의 `api.policySave` 경로 | [명세](../../api/operation/cert/01-cert.md) |
| 삭제 | mixin의 `api.policyDelete` 경로 | [명세](../../api/operation/cert/01-cert.md) |

Mixin 기반(`mixins/unlicensed.js`, `mixins/unlicensedDetail.js`)으로 구현. 목록/상세/등록/수정/삭제 로직이 mixin에 분리되어 있고, 실제 컴포넌트는 UI 템플릿만 담당.
