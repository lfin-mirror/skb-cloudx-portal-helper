# 네트워크 보안 정책 (A050601/T050601, T050611)

상세 정책(A0506) 하위의 네트워크 관련 메뉴를 통합 정리. 예외 네트워크, 접근 통제(보안그룹).

## 메뉴 관계

```
예외 네트워크 (A050601/T050601)  ← 2차 인증을 면제할 IP 대역 그룹 정의
     │
     │  인증 정책 / 가상 PC 정책에서 참조
     │
접근 통제 (T050611)              ← OpenStack 보안그룹 기반 네트워크 허용/차단 규칙 정의
     │                              v2.2.11에서 기존 접근 차단(A050602/T050602) 대체
     │
     │  가상 PC 정책(T0503)에서 선택
```

### 비유

```
예외 네트워크 = "사내 IP라서 2차 인증 건너뛰기" 허용 목록
접근 통제    = "이 보안그룹의 이 프로토콜/포트는 허용 또는 차단" 규칙 세트
```

## 예외 네트워크 (A050601)

- 경로: `/policy/policy-manage/excerpt-network`
- 컴포넌트: `ExcerptNetwork.vue`

2차 인증을 **면제**할 네트워크(IP 대역) 그룹을 정의. 인증 정책(A0501/A0502)이나 가상 PC 정책(A0503)에서 이 그룹을 참조하면 해당 IP에서 접속 시 2차 인증 생략.

### UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조. 행 클릭 시 좌측에 상세/수정 폼 표시.

|------|------|
| 정책명 | `exc_nw_grp_nm` |
| 생성자 | `reg_conn_id` |
| 생성일시 | `reg_ts` |
| 수정자 | `mod_conn_id` |

그룹 안에 여러 IP 대역을 추가하는 방식. 예: "본사 네트워크" 그룹에 `10.0.1.0/24`, `10.0.2.0/24` 등록.

## 접근 통제 — 보안그룹 (T050611)

TA 전용 (SA 메뉴에 없음). IAADEV-7824, IAADEV-7825, IAADEV-7887.

- 경로: `/policy/policy-manage/security-group`
- 컴포넌트: `SecurityGroup.vue`
- 하위 컴포넌트: `SecurityGroupCreate.vue`, `SecurityGroupDetail.vue`, `SecurityGroupRuleAddModal.vue`

OpenStack 보안그룹 기반 네트워크 접근 통제 규칙 관리. 가상 PC 정책(T0503)에서 보안그룹을 선택하여 VM에 적용.

### UI 레이아웃

상단 목록 + 하단 상세(Split Panel) 구조.

- **목록 화면**: 검색 + 테이블 + 페이지네이션

#### 탭 2: Rule

보안그룹에 속한 규칙 목록 조회 및 관리.

| 컬럼 | 필드 | 정렬 | 표시값 |
|------|------|------|--------|
| 조치 (`action`) | `action` | O | `allow` → 허용, `deny` → 차단 |
| 방향 (`direction`) | `direction` | O | `egress` → 내보냄, `ingress` → 들어옴 |
| IP 프로토콜 (`protocol`) | `protocol` | O | `any`/null/0 → 전체, 나머지 대문자 변환 |
| 포트 범위 (`port_range_min`~`port_range_max`) | `port_range_max` | O | 단일 포트는 Well-Known 포트명 병기 (SSH, HTTP, RDP 등). ICMP는 type-code 표시 |
| 원격 IP 대역 (`remote_ip_prefix`) | `remote_ip_prefix` | O | 없으면 `-` |
| 설명 (`description`) | `description` | O | 없으면 `-` |
| 삭제 | — | X | 행별 삭제 버튼. `DELETE /v1/resource/policies/security-group/{id}/rule/{ruleId}` 호출 |

| 버튼 | 동작 |
|------|------|
| 항목 추가 | Rule 추가 모달(`SecurityGroupRuleAddModal`) 표시 |

**포트 범위 표시 규칙:**

- protocol이 `any`/null/0 → "전체"
- `port_range_min`=1, `port_range_max`=65535 → "전체"
- `port_range_min` == `port_range_max` → 단일 포트. Well-Known 포트(22=SSH, 80=HTTP, 443=HTTPS, 3389=RDP 등) 병기
- ICMP이고 type/code가 -1/-1 → "전체"
- 그 외 → `{min}-{max}` 범위 표시

### Rule 추가 모달 (SecurityGroupRuleAddModal)

상세 화면 Rule 탭의 "항목 추가" 버튼으로 표시되는 다이얼로그.

#### 폼 필드

| 필드 | 코드 필드 | 필수 | 설명 |
|------|----------|------|------|
| 조치 (`action`) | `action` | O | 허용(`allow`) / 차단(`deny`) |
| 규칙 (`ruleMenu`) | `ruleMenu` | O | 사용자 정의 규칙(TCP/UDP/ICMP/다른 프로토콜) + 사전 정의 규칙(DNS, HTTP, HTTPS, SSH, RDP 등 16종) |
| 설명 (`description`) | `description` | X | 최대 255자 |
| 방향 (`direction`) | `direction` | 조건부 | 들어옴(`ingress`) / 내보냄(`egress`). 사용자 정의 규칙 + 모든 TCP/UDP/ICMP 선택 시 표시. 사전 정의 규칙은 `ingress` 고정 |
| 포트 유형 (`portOrRange`) | `portOrRange` | 조건부 | 포트 / 포트 범위 / 모든 포트. 사용자 정의 TCP/UDP에서만 표시 |
| 포트 (`port`) | `port` | 조건부 | 1~65535. 포트 유형이 "포트"일 때 표시 |
| From Port (`fromPort`) | `fromPort` | 조건부 | 1~65535. 포트 유형이 "포트 범위"일 때 표시 |
| To Port (`toPort`) | `toPort` | 조건부 | 1~65535. From Port 이상이어야 함. 포트 유형이 "포트 범위"일 때 표시 |
| 타입 (`icmpType`) | `icmpType` | 조건부 | 0~255. 사용자 정의 ICMP에서만 표시. 빈 값 = 전체 |
| 코드 (`icmpCode`) | `icmpCode` | 조건부 | 0~255. 사용자 정의 ICMP에서만 표시. 빈 값 = 전체 |
| IP Protocol (`ipProtocol`) | `ipProtocol` | 조건부 | -1~255. "다른 프로토콜" 선택 시 표시. 빈 값/-1 = 전체 |
| 원격 IP 유형 (`remote`) | `remote` | O | CIDR 고정 |
| 원격 IP 대역 (`cidr`) | `cidr` | O | CIDR 형식 (예: `0.0.0.0/0`). IPv4/IPv6 검증 |

#### 규칙 유형별 필드 표시 조건

| 규칙 유형 | 방향 | 포트 유형 | 포트/범위 | ICMP | IP Protocol |
|-----------|------|-----------|-----------|------|-------------|
| 사용자 정의 TCP/UDP | O | O | O | X | X |
| 사용자 정의 ICMP | O | X | X | O | X |
| 다른 프로토콜 | O | X | X | X | O |
| 모든 TCP/UDP/ICMP | O | X | X | X | X |
| 사전 정의 규칙 (DNS, HTTP 등) | X | X | X | X | X |

#### 사전 정의 규칙 목록

코드상 `ruleOptions.custom` 배열에 속하며 `ruleType === 'all'`로 분류되는 3종. 방향(`direction`) 선택 가능, 포트/ICMP 입력 없음 — 사용자 정의와 사전 정의의 중간 성격.

| 규칙명 | 프로토콜 | 포트 | 코드 분류 |
|--------|----------|------|-----------|
| 모든 ICMP | ICMP | 전체 | `custom` / `ruleType: 'all'` |
| 모든 TCP | TCP | 1-65535 | `custom` / `ruleType: 'all'` |
| 모든 UDP | UDP | 1-65535 | `custom` / `ruleType: 'all'` |

`ruleOptions.preset` 배열에 속하는 사전 정의 규칙 14종. 방향은 `ingress` 고정, 프로토콜/포트 입력 없음.

| 규칙명 | 프로토콜 | 포트 |
|--------|----------|------|
| DNS | TCP | 53 |
| HTTP | TCP | 80 |
| HTTPS | TCP | 443 |
| IMAP | TCP | 143 |
| IMAPS | TCP | 993 |
| LDAP | TCP | 389 |
| MS SQL | TCP | 1433 |
| MYSQL | TCP | 3306 |
| POP3 | TCP | 110 |
| POP3S | TCP | 995 |
| RDP | TCP | 3389 |
| SMTP | TCP | 25 |
| SMTPS | TCP | 465 |
| SSH | TCP | 22 |
