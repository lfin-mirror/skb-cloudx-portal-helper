# 네트워크 보안 정책 (A050601/T050601, A050602/T050602, A050611/T050611)

상세 정책(A0506) 하위의 네트워크 관련 3개 메뉴를 통합 정리. 예외 네트워크, 접근 차단, 보안 그룹.

## 3개 메뉴의 관계

```
예외 네트워크 (A050601)     ← 2차 인증을 면제할 IP 대역 그룹 정의
     │
     │  인증 정책 / 가상 PC 정책에서 참조
     │
접근 차단 (A050602)         ← 서비스별 네트워크 차단/허용 규칙 정의
     │
     │  가상 PC 정책(A0503)에서 선택
     │
보안 그룹 (A050611)         ← OpenStack Security Group 동기화 + CRUD
                               VM 레벨 방화벽 규칙 (Neutron)
```

### 비유

```
예외 네트워크 = "사내 IP라서 2차 인증 건너뛰기" 허용 목록
접근 차단    = "이 서비스의 이 포트는 막기" 규칙 세트
보안 그룹    = VM마다 붙는 방화벽 규칙 (OpenStack Neutron 연동)
```

## 예외 네트워크 (A050601)

- 경로: `/policy/policy-manage/excerpt-network`
- 컴포넌트: `ExcerptNetwork.vue`

2차 인증을 **면제**할 네트워크(IP 대역) 그룹을 정의. 인증 정책(A0501/A0502)이나 가상 PC 정책(A0503)에서 이 그룹을 참조하면 해당 IP에서 접속 시 2차 인증 생략.

### UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조. 행 클릭 시 좌측에 상세/수정 폼 표시.

### 테이블 컬럼

| 컬럼 | 필드 |
|------|------|
| 정책명 | `exc_nw_grp_nm` |
| 생성자 | `reg_conn_id` |
| 생성일시 | `reg_ts` |
| 수정자 | `mod_conn_id` |

### API

| 동작 | API | 명세 |
|------|-----|------|
| 목록 | `GET /v1/operation/policy/excn/excngrp` | [명세](../../api/operation/excerpt-network/01-excerpt-network.md) |
| 상세 | `GET /v1/operation/policy/excn/excngrp/{id}` | [명세](../../api/operation/excerpt-network/01-excerpt-network.md) |
| 생성 | `POST /v1/operation/policy/excn/excngrp` | [명세](../../api/operation/excerpt-network/01-excerpt-network.md) |
| 수정 | `PUT /v1/operation/policy/excn/excngrp/{id}` | [명세](../../api/operation/excerpt-network/01-excerpt-network.md) |
| 삭제 | `DELETE /v1/operation/policy/excn/excngrp/{id}` | [명세](../../api/operation/excerpt-network/01-excerpt-network.md) |

### 상세 폼 (ExcerptNetworkSetting)

그룹 안에 여러 IP 대역을 추가하는 방식. 예: "본사 네트워크" 그룹에 `10.0.1.0/24`, `10.0.2.0/24` 등록.

## 접근 차단 (A050602)

- 경로: `/policy/policy-manage/access-block`
- 컴포넌트: `AccessBlock.vue`

서비스별로 네트워크 접근을 차단/허용하는 규칙 세트. 가상 PC 정책(A0503)의 `acc_blck_plcy_id` 드롭다운에서 선택하여 VM에 적용.

### UI 레이아웃

좌측 폼(8칸) + 우측 리스트(16칸) 2단 구조. 예외 네트워크와 동일한 패턴.

### 테이블 컬럼

| 컬럼 | 필드 |
|------|------|
| 정책명 | `acc_blck_plcy_nm` |
| 생성자 | `reg_conn_id` |
| 생성일시 | `reg_ts` |
| 수정자 | `mod_conn_id` |
| 수정일시 | `mod_ts` |

### 상세 폼 (AccessBlockSetting)

서비스 목록 테이블에서 각 서비스별로 차단 여부를 설정:

| 컬럼 | 설명 |
|------|------|
| `acc_blck_serv_nm` | 서비스명 |
| `ip` / `port` | 서비스 IP와 포트 |
| `tot_net_blck_yn` | 전체 네트워크에서 차단 여부 |
| `exc_net_blck_yn` | 예외 네트워크에서 차단 여부 |

| 동작 | API | 명세 |

| 상세 | `GET /v1/operation/policys/acclbck/plcys/{id}` | [명세](../../api/operation/access-control/01-access-control.md) |

## 보안 그룹 (A050611, T050611)

경로: /policy/policy-manage/security-group

컴포넌트: views/policy/policyManage/SecurityGroup.vue

SA/TA 공통. OpenStack Neutron의 Security Group을 관리. VM 레벨 방화벽 규칙 — 특정 포트/프로토콜의 인바운드/아웃바운드 트래픽 허용/차단.

### SA vs TA 차이

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| **메뉴 존재** | — | SA 메뉴 트리에 없음 (가상 PC 정책에서 참조만) | 상세 정책 하위 메뉴로 존재 |
| **CRUD** | — | — | CRUD + Sync |
| **Sync** | — | — | OpenStack에서 동기화 |

SA는 보안 그룹 메뉴 자체가 없고, 가상 PC 정책(A0503)의 보안 항목 테이블에서 `sg_grp_id` 드롭다운으로 참조만 함 (SA에서는 `forceDisabled`).

### UI 레이아웃

Split Panel (상하 분할). 상단 리스트 + 하단 상세. 행 클릭 시 하단에 상세 패널 표시.

### 테이블 컬럼

| 컬럼 | 필드 |
|------|------|
| 보안그룹명 | `name` |
| 설명 | `description` |
| 생성자 | `reg_conn_id` |
| 생성일시 | `reg_ts` |
| 삭제 | 버튼 (행별) |

### 버튼 동작

| 버튼 | 클릭 시 동작 |
|------|------------|
| **등록** | 보안 그룹 생성 → `POST /v1/resource/policies/security-group` |
| **삭제** (행별) | 확인 팝업 → `DELETE /v1/resource/policies/security-group/{id}` |
| **Sync** | OpenStack Neutron에서 최신 Security Group 동기화 → `POST /v1/resource/policies/security-group/sync` |

### default 보안 그룹

OpenStack은 모든 프로젝트(테넌트)에 `default` 보안 그룹을 자동 생성. 삭제해도 자동 재생성됨.

### API

| 동작 | API | 명세 |
|------|-----|------|
| 목록 | `GET /v1/resource/policies/security-group` | [명세](../../api/resource/security-group/01-security-group.md) |
| 생성 | `POST /v1/resource/policies/security-group` | [명세](../../api/resource/security-group/01-security-group.md) |
| 삭제 | `DELETE /v1/resource/policies/security-group/{id}` | [명세](../../api/resource/security-group/01-security-group.md) |
| 동기화 | `POST /v1/resource/policies/security-group/sync` | [명세](../../api/resource/security-group/01-security-group.md) |

### 다른 메뉴와의 차이

예외 네트워크/접근 차단은 CloudX 자체 정책 (`app-ms-operation`). 보안 그룹은 OpenStack Neutron 연동 (`app-ms-resource`).

```
예외 네트워크 / 접근 차단 → /v1/operation/...  (app-ms-operation)
보안 그룹                → /v1/resource/...   (app-ms-resource → OpenStack)
```
