# 백업/스냅샷 정책 (A0505, T0505)

경로: /policy/backup-snapshot-policy

SA 컴포넌트: `BackupSnapshotSupadm.vue`

TA 컴포넌트: `BackupSnapshot.vue`

VM 디스크의 자동 백업과 스냅샷 스케줄을 정의하는 정책.

## UI 레이아웃

SA: 폼 기반 단일 화면 — 목록에서 첫 번째 정책을 자동 로드하여 폼에 표시. 등록/삭제 버튼 없음.

TA: 목록 + 상세 폼. 정책 등록/삭제/가져오기 가능.

## 폼 필드

### 백업 설정

| 화면 라벨 (i18n) | 필드 | 타입 | disabled 조건 |
|-----------------|------|------|-------------|
| 백업/스냅샷 정책명 (`page.policy.label.backupSnapshotPolicyName`) | `bkup_snap_plcy_nm` | 텍스트 (readonly) | — |
| 자동 백업 설정 (`page.policy.label.autoBackupSetting`) | `auto_bkup_yn` | 셀렉트 (사용/미사용) | — |
| | ↓ **auto_bkup_yn = 'N'일 때 하위 필드 disabled** | | |
| 최대 백업 보관 수 (`page.policy.label.backupMaxNumSelect`) | `max_bkup_cnt` | InputNumber (min: 0) | `auto_bkup_yn == 'N'` |
| 자동 백업 주기 (`page.policy.label.autoBackupPeriod`) | `bkup_prid_cd` | 라디오 (U027D/W/M) | `auto_bkup_yn == 'N'` |
| | ↓ **bkup_prid_cd = 'U027W'일 때만 활성** | | |
| 　요일 | `bkup_set_dw_cd` | 셀렉트 (코드 U024) | `auto_bkup_yn == 'N'` 또는 `bkup_prid_cd != 'U027W'` |
| | ↓ **bkup_prid_cd = 'U027M'일 때만 활성** | | |
| 　날짜 | `bkup_set_dd` | 셀렉트 (1/5/10/15/20/25일) | `auto_bkup_yn == 'N'` 또는 `bkup_prid_cd != 'U027M'` |
| 　시간 | `time_cd.bkup_{index}` | 셀렉트 (01~24시) | 선택한 주기와 일치하는 index만 활성 |

### 스냅샷 설정

백업 설정과 동일한 구조. 필드 접두사가 `snap_`으로 변경.

| 화면 라벨 (i18n) | 필드 | disabled 조건 |
|-----------------|------|-------------|
| 자동 스냅샷 설정 (`page.policy.label.autoSnapshotSetting`) | `auto_snap_yn` | — |
| | ↓ **auto_snap_yn = 'N'일 때 하위 필드 disabled** | |
| 최대 스냅샷 보관 수 (`page.policy.label.snapshotMaxNumSelect`) | `max_snap_cnt` | `auto_snap_yn == 'N'` |
| 자동 스냅샷 주기 (`page.policy.label.autoSnapshotPeriod`) | `snap_prid_cd` | `auto_snap_yn == 'N'` |

### 백업 vs 스냅샷

```
백업:   VM 디스크 전체를 별도 저장소에 복사 → 복원용 (느리지만 안전)
스냅샷: 디스크의 특정 시점 상태를 기록 → 빠른 롤백용 (CoW 방식)
```

두 설정은 독립적 — 백업만 켜거나, 스냅샷만 켜거나, 둘 다 가능.

## 버튼 동작

| 버튼 | SA | TA | 클릭 시 동작 |
|------|----|----|------------|
| **저장** | 노출 | 노출 | 등록: `POST /v1/operation/policy/bkupsnap/grps` / 수정: `PUT /v1/operation/policy/bkupsnap/info/{id}` |
| **삭제** | 비노출 | 노출 (`id && !isSuperAdmin`) | 확인 팝업 → `DELETE /v1/operation/policy/bkupsnap/info/{id}` |

| **정책 가져오기** | 비노출 | 노출 (`!id && !isSuperAdmin`) | SA 템플릿(U023S) 복사 |

## SA vs TA 차이

| 항목 | 세부 항목 | SA | TA |
|------|----------|----|----|
| **대상 코드** | `bkup_snap_plcy_tgt_cd` | `U023S` | `U023T` |
| **UI 구조** | — | 폼 1개 직접 편집 | 목록 + 상세 폼 |
| **역할** | — | 기본 정책 템플릿 정의 | 테넌트별 정책 커스터마이징 |
| **등록** | — | 비노출 | 노출 (`!isSuperAdmin`) |
| **삭제** | — | 비노출 | 노출 (`id && !isSuperAdmin`) |
| **정책 가져오기** | — | 비노출 | 노출 (SA 템플릿 복사) |
| **테이블 컬럼** | 정책명 (`bkup_snap_plcy_nm`) | 표시 | 표시 |
| | 정책 타입 (`bkup_snap_plcy_tgt_cd_nm`) | 표시 | 비노출 |
| | 풀 수 (`pool_cnt`) | 비노출 | 표시 (클릭 시 모달) |
| | VM 수 (`vm_cnt`) | 비노출 | 표시 (클릭 시 모달) |
