# 가상 디스크 (T1301)

경로: /virtual-disk/virtual-disk-manage

컴포넌트: views/virtualDisk/VirtualDiskManage.vue

모달: views/virtualDisk/components/VirtualDiskModal.vue

Mixin: views/virtualDisk/mixins/index.js

**TA 전용 메뉴** (SA에 없음). 개인 디스크(로컬 디스크)의 연결/분리/백업/복구/삭제 관리. `isSuperAdmin` 분기 없음.

## 연관 메뉴/개념

```
[시스템 자원 > 스토리지]           ← 디스크 "타입과 현황" 관리 (인프라 레벨)
├── 볼륨 타입 (A100501/T100501)     디스크 종류 정의 (NetApp 백엔드, QoS)
├── 디스크 QoS (A100502)            디스크 I/O 제한 정책 (SA 전용)
└── 볼륨 자원 (A100503/T100503)     생성된 디스크 현황 조회

[테넌트 설정]                     ← 어떤 볼륨 타입을 쓸지 "할당"
├── 볼륨 타입 설정                  이 테넌트가 쓸 수 있는 볼륨 타입 할당
├── 개인디스크 볼륨 설정             개인 디스크용 볼륨 타입 선택
└── 개인디스크 볼륨 스케쥴링 정책    개인 디스크 자동 할당 방식

[가상 디스크] (T1301)             ← 개인 디스크의 "운영" (이 메뉴)
    연결/분리/백업/복구/삭제
```

비유:
```
스토리지     = "어떤 종류의 USB 메모리를 구매 가능한지" (128GB SSD, 256GB HDD 등)
테넌트 설정  = "우리 부서는 128GB SSD만 쓸 수 있게 허용"
가상 디스크  = "이 USB 메모리를 누구 PC에 꽂고, 빼고, 백업하는 작업"
```

OpenStack 관점에서는 Cinder Volume의 Attach/Detach/Backup/Restore에 대응.

## 개념

VM에는 기본 디스크(OS 디스크)가 있고, 추가로 **개인 디스크**(영구 디스크)를 붙일 수 있음. 이 메뉴에서 개인 디스크의 생명주기를 관리.

```
VM (가상 PC)
├── OS 디스크 (기본, 볼륨 타입에 따라 자동 생성)
└── 개인 디스크 (선택, 이 메뉴에서 관리)
    ├── 연결 (attach): VM에 디스크 연결
    ├── 분리 (detach): VM에서 디스크 분리
    ├── 백업 (backup): 디스크 백업 생성
    ├── 복구 (restore): 백업에서 복원
    └── 삭제 (delete): 디스크 삭제
```

## UI 레이아웃

리스트 + 모달 구조. 테이블에서 디스크 선택 후 드롭다운 또는 모달로 작업 실행.

### 필터

디스크 상태 (`dsk_sts_cd`)로 필터링.

### 테이블 컬럼

| 컬럼 | 필드 | 설명 |
|------|------|------|
| 테넌트명 | `tnt_nm` | — |
| 사용자그룹명 | `usr_grp_nm` | — |
| 디스크명 | `dsk_nm` | — |
| 가상PC명 | `vm_nm` | — |
| 가상PC상태 | `vm_power_sts_cd` | 상태 아이콘 (`tableToStatusItem`) |
| 용량(GB) | `size` | — |
| 사용자명 | `acct_nm` | — |
| 사용자ID | `acct_conn_id` | — |
| 디스크상태 | `dsk_sts_cd` | 상태 아이콘 (`tableToStatusItem`) |
| 생성자ID | `reg_conn_id` | — |
| 생성일 | `reg_ts` | — |

### 버튼

일괄 작업 (다중 선택):

| 버튼 | 값 | 설명 |
|------|-----|------|
| 분리 | `multi-detach` | 선택한 디스크 일괄 분리 |
| 삭제 | `multi-delete` | 선택한 디스크 일괄 삭제 |

개별 작업 (행별 드롭다운):

| 버튼 | 값 | 설명 |
|------|-----|------|
| 연결 | `attach` | VM에 디스크 연결 |
| 분리 | `detach` | VM에서 디스크 분리 |
| 백업 | `backup` | 디스크 백업 생성 |
| 복구 | `restore` | 백업에서 복원 |
| 삭제 | `delete` | 디스크 삭제 |

### 버튼 비활성화 조건 (`conditionDiskCompare`)

디스크 상태(`dsk_sts_cd`)와 VM 전원 상태(`vm_power_sts_cd`)에 따라 각 버튼 disabled 여부 결정:

| 작업 | disabled 조건 | 이유 |
|------|-------------|------|
| 연결 (attach) | `dsk_sts_cd = 'V009USE'` (사용중) | 이미 VM에 연결된 디스크 |
| 분리 (detach) | `dsk_sts_cd = 'V009AVL'` (사용가능) + VM 꺼짐 | 연결되지 않은 디스크이거나 VM 꺼져있음 |
| 백업 (backup) | `dsk_sts_cd = 'V009AVL'` + VM 없음, 또는 `dsk_sts_cd = 'V009USE'` + `vm_power_sts_cd = 'V002ONC'` | VM 전원 켜진 상태에서 백업 불가 |
| 복구 (restore) | `dsk_sts_cd = 'V009AVL'` + `vm_power_sts_cd = 'V002ONC'` | VM 전원 켜진 상태에서 복구 불가 |
| 삭제 (delete) | `dsk_sts_cd = 'V009USE'` (사용중) | VM에 연결된 디스크는 삭제 불가 |

## 모달 — eventType별 UI 분기 (VirtualDiskModal)

### 연결 (attach)

- 안내 메시지 (alert)
- 사용자ID, 사용자명, 디스크명 (읽기 전용)
- VM 목록 테이블 — VM 선택 후 `선택` 버튼

### 분리 (detach)

- 안내 메시지 (alert)
- 사용자ID, 사용자명, 디스크명 (읽기 전용)
- `분리` 버튼

### 백업 (backup)

- 가이드 문구 (disk.message.guideOne, guideTwo)
- 사용자ID, 사용자명, 디스크명 (읽기 전용)
- 백업 디스크명 (입력 필드)
- `생성` 버튼

### 복구 (restore) — 모달 너비 760px (다른 eventType은 660px)

- 안내 메시지 (alert)
- 사용자ID, 사용자명 (읽기 전용)
- 백업 목록 테이블 — `cre_mtd_cd_nm`(생성방식), `bkup_dsk_nm`(백업명), `reg_ts`(생성일) 컬럼
- 체크박스 선택 후 `복구` 또는 `제거` 버튼
