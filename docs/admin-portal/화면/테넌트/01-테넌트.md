# 테넌트 (A0802, T0802)

경로: /tenant/index

컴포넌트: views/tenant/Index.vue

상세: views/tenant/IndexDetail.vue

SA/TA 공통 메뉴. CloudX에서 가장 중요한 리소스 단위 — 테넌트가 Zone, 네트워크, 스토리지, 사용자, VM 모든 것의 소유 주체.

## 테넌트가 뭔가

```
테넌트 = "이 조직(회사/부서)의 VDI 환경 전체"

테넌트에 묶이는 것:
  ├── Zone → 물리 호스트 영역 (CV_TNT_M.ZONE_NM)
  ├── 네트워크 → 가상 네트워크/서브넷/라우터
  ├── 볼륨 타입 → 스토리지 프로필
  ├── 사용자 그룹 → 조직 구조
  ├── 가상 PC 그룹/풀 → VM 관리 단위
  ├── 정책 → 인증/보안/백업 정책
  └── TA 관리자 → 이 테넌트를 운영하는 관리자
```

## 호스트 설정 (`host_tnt_yn`) — 리소스 격리 수준 결정

멀티테넌트 환경에서 물리 호스트를 어떻게 할당할지 결정하는 핵심 설정.

| 값 | 모드 | 의미 |
|----|------|------|
| `Y` | 호스트 사용 | 테넌트에 특정 호스트(물리 서버)를 직접 지정. VM은 해당 호스트에서만 생성 |
| `N` | 호스트 미사용 | Zone(가용 영역) 단위로 할당. 플랫폼이 Zone 내 호스트를 자동 배치 |

### 테넌트 생성/수정 시 API 분기 (IndexDetail.vue:1804)

```
host_tnt_yn === 'Y'
  → POST /v1/resource/tenants/manager/usehost (host_ids 포함)

host_tnt_yn === 'N'
  → POST /v1/resource/tenants/manager/nonehost (zone_nm 포함)
```

### 상세 UI 분기 (IndexDetail.vue:63-83)

```
host_tnt_yn === 'Y':
  ├── 호스트 목록 표시 (호스트 선택 모달)
  ├── Max CPU / Max Memory 제한값 설정
  └── HA 실행 모드 선택 가능 (TA에게 노출)

host_tnt_yn === 'N':
  └── Zone 선택 UI 표시 (Zone 드롭다운)
      → HA는 자동(Y)으로 고정 (TA가 변경 불가)
```

### 다른 화면에 미치는 영향

| 영향 받는 화면 | `Y` (호스트 사용) | `N` (호스트 미사용) |
|-------------|-----------------|-------------------|
| Zone 관리 (Zone.vue:159) | 호스트 설정 버튼 표시 | 호스트 설정 버튼 숨김 |
| 가상 PC 목록 (VirtualPcGroup.vue:914) | SearchFilter에 호스트 필터 노출 | 호스트 필터 숨김 |
| HA 실행 (IndexDetail.vue:1894) | TA가 HA 모드 선택 가능 | HA 자동(Y) 고정 |
| Zone 조회 쿼리 (ZoneMapper.xml) | `CP_ZONE_M.TNT_ID`로 직접 매핑 | `CV_TNT_M.ZONE_NM`으로 이름 매핑 |

### 비유

```
host_tnt_yn = 'Y' (호스트 사용):
  → "이 서버 랙은 우리 회사 전용. 다른 회사 VM 못 올라감"
  → 보안 요구사항 높은 환경 (금융/공공)

host_tnt_yn = 'N' (호스트 미사용):
  → "이 Zone(구역) 안에서 플랫폼이 알아서 배치해줘"
  → 일반 기업 환경
```

"호스트 테넌트"는 OpenStack 공식 개념이 아닌 CloudX 자체 확장.
