## 추천 읽기 순서

### 1단계: 전체 구조 파악

**[08-api-흐름-요약](./08-api-흐름-요약.md)** 먼저 읽기. admin-portal에서 OpenStack까지 4단계 호출 경로(`admin-portal → app-ms-resource → plat-ms-vid4o → OpenStack`)를 이해하면 이후 문서가 쉬워짐. 어떤 메뉴가 어떤 OpenStack 서비스를 쓰는지 한눈에 파악 가능.

### 2단계: 가장 큰 개념부터

**[01-keystone-도메인과-테넌트](./01-keystone-도메인과-테넌트.md)** → 모든 리소스의 소유 구조. Domain > Project(Tenant) 계층이 SA/TA 역할 분리의 근간.

### 3단계: 컴퓨트 (VM이 실행되는 곳)

**[02-nova-호스트와-zone](./02-nova-호스트와-zone.md)** → VM이 어디서 돌아가는지. Host(물리서버), Zone(가용영역), Flavor(사양) 개념.

**[03-nova-마이그레이션과-HA](./03-nova-마이그레이션과-HA.md)** → VM을 왜, 어떻게 옮기는지. Migration(정상 이동) vs Evacuate(장애 복구) 차이.

### 4단계: 네트워크 (VM이 통신하는 방법)

**[04-neutron-네트워크](./04-neutron-네트워크.md)** → 물리/가상 네트워크, 서브넷, 라우터, QoS. 네트워크 메뉴 4개(네트워크/서브넷/라우터/QoS)가 전부 여기서 설명.

**[07-proxy-vdi-네트워크](./07-proxy-vdi-네트워크.md)** → VDI 전용 네트워크 구성. OpenStack 기능이 아닌 CloudX 인프라.

### 5단계: 스토리지 (VM의 데이터가 저장되는 곳)

**[05-cinder-블록-스토리지](./05-cinder-블록-스토리지.md)** → 볼륨(가상 디스크), 볼륨 타입, QoS. 스토리지 메뉴 3개(볼륨타입/디스크QoS/볼륨자원)가 여기.

**[06-nas-공유-스토리지](./06-nas-공유-스토리지.md)** → 공유 폴더. Block Storage와의 차이. NetApp 연동.

### 6단계: 이미지 (VM의 OS가 어디서 오는가)

**[09-glance-이미지](./09-glance-이미지.md)** → 베이스 이미지/골든 이미지의 실체. Cinder 볼륨과의 변환 관계 (이미지→볼륨→스냅샷). Copy-on-Write로 VM 대량 생성 시 스토리지 절감.

05(Cinder)를 먼저 읽고 09(Glance)를 읽으면 이미지↔볼륨↔스냅샷 변환 관계 이해가 쉬움.

## 한 줄 요약

```
Keystone(누가 소유) → Nova(어디서 실행) → Neutron(어떻게 통신) → Cinder/NAS(어디에 저장)
```
