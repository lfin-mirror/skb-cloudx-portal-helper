## 사전 지식

정책 문서를 읽기 전에 아래 개념이 필요:

- **Cloud PC 보안 정책 구조** (일반/예외 네트워크, 7개 보안 항목, API 응답 형태) → [Cloud PC 보안 정책](../../../term/01-virtual-pc-policy.md) 참고
- **VDI / 하이퍼바이저 / KVM / QEMU** 등 기본 용어 → [VDI 인프라 용어](../../../term/02-vdi-infra-terms.md) 참고
- **VM이 어디서 실행되는지** (호스트, Zone) → [시스템 자원 읽기 가이드](../시스템%20자원/00-읽기-가이드.md) 참고

## 읽기 순서

### 1단계: VDI 원격 접속의 큰 그림

**[01-spice-프로토콜](./01-spice-프로토콜.md)** — "VDI에서 SPICE의 역할" 섹션부터 읽기. 사용자 PC ↔ VM 연결 구조, Metadata 정책·USB 정책·가상 PC 정책이 어떻게 연결되는지 파악.

### 2단계: 프로토콜 비교

같은 문서의 "VNC/RDP와의 비교" 섹션. VNC·RDP·SPICE 차이와 CloudX가 SPICE를 선택한 이유.

### 3단계: SPICE 설정 상세

같은 문서의 나머지 섹션. 압축 옵션(Image/JPEG/zlib), Streaming Mode, USB Redirection 등 Metadata 정책 화면의 각 설정이 실제로 무엇을 제어하는지 확인.

   압축 옵션의 원리                            각 필드의 값과 UI 동작
   USB Redirection 원리                       USB 설정 (usb_version, usb_redirection)
                                             06-USB 정책.md
                                               USB 유형/제조사/모델 CRUD

OpenStack 지식이 필요한 설정을 만나면 이 폴더의 문서를, UI 동작이 궁금하면 SA/정책/ 문서를 참고.
