# AD (Active Directory)

Microsoft의 디렉터리 서비스로, 기업 내 사용자 계정과 비밀번호를 중앙에서 관리한다. CloudX에서 AD 연동이 활성화된 테넌트는 비밀번호 관리를 AD에 위임한다. CloudX 자체에서 비밀번호를 변경하거나 초기화하지 않고, AD에서 처리하는 구조다.

## 관련 설정값

AD 연동과 관련된 값은 테넌트 레벨과 사용자 레벨로 나뉜다. 테넌트가 AD를 사용하더라도 모든 사용자가 AD 계정인 것은 아니다.

### 테넌트 레벨

`permission.js`의 `tntExists()` 응답에서 받아 sessionStorage + localStorage에 저장된다.

| 키 | 값 | 설명 |
|---|---|---|
| `adItlkUsgYn` | `'true'` / `'false'` | 테넌트의 AD 연동 사용 여부 |
| `adModYn` | `'true'` / `'false'` | AD 수정 모드 — `'true'`: 읽기쓰기(CloudX에서 AD에 비밀번호 쓰기 가능), `'false'`: 읽기전용 |

### 사용자 레벨

로그인 API 응답(`POST /v1/gw/authentications/`)에 포함된다.

| 키 | 값 | 설명 |
|---|---|---|
| `ad_itlk_acct_yn` | `'Y'` / `'N'` | 개별 사용자가 AD 연동 계정인지 여부 |

## 프론트엔드 영향

로그인 API 호출과 폼 UI는 AD 연동 여부와 무관하게 동일하다. 백엔드(app-ms-resource)가 AD 연동 테넌트의 사용자라면 AD 서버에 인증을 위임하고, 아니라면 자체 DB로 인증한다. 프론트엔드는 이 차이를 알지 못한다.

AD 설정값이 화면에 미치는 영향의 상세 내용은 [user-portal/auth/03-account-recovery-and-ad.md](../auth/03-account-recovery-and-ad.md)에서 다룬다.
