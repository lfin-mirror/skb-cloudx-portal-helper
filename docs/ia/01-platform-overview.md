# 플랫폼 개요

CloudX VDI 플랫폼의 전체 흐름과 역할별 관계 정리.

## 플랫폼 흐름

```
SA (인프라 구축)          TA (테넌트 운영)          사용자 (VM 접속)
──────────────          ──────────────          ──────────────
도메인/Zone 생성          │                        │
호스트 등록               │                        │
네트워크/스토리지 구성     │                        │
템플릿(Flavor+Image) 정의 │                        │
테넌트 생성 + 리소스 할당 ─▶ 테넌트 설정 조정         │
정책 템플릿 정의 ─────────▶ 정책 가져오기/커스터마이징  │
                         사용자 계정 생성 ──────────▶ 로그인
                         가상 PC 풀/그룹 구성        │
                         VM 프로비저닝 ─────────────▶ Cloud PC 접속
                         모니터링/장애 대응 ◀─────── 장애 신청
```

## 포털 간 관계

```
┌─────────────────────────────────────────────────┐
│                  admin-portal                    │
│                                                  │
│   ┌──────────────┐    ┌──────────────┐          │
│   │  SA 화면      │    │  TA 화면      │          │
│   │              │    │              │          │
│   │ 인프라 구축   │───▶│ 테넌트 운영   │          │
│   │ 정책 템플릿   │───▶│ 정책 적용     │          │
│   │ 테넌트 관리   │───▶│ 사용자/VM 관리│          │
│   └──────────────┘    └──────┬───────┘          │
│                              │ 콘텐츠 관리        │
└──────────────────────────────┼───────────────────┘
                               │ (공지/FAQ/매뉴얼/팝업)
                               ▼
                    ┌──────────────────┐
                    │   user-portal     │
                    │                  │
                    │ Cloud PC 접속    │
                    │ 디스크/스냅샷     │
                    │ 고객지원          │
                    └──────────────────┘
```

- SA와 TA는 동일 admin-portal 코드베이스. `isSuperAdmin` 분기로 UI 제어.
- TA가 admin-portal에서 등록한 공지/FAQ/매뉴얼이 user-portal에 노출.
- user-portal은 멀티테넌트 라우트(`/:tenant`)로 테넌트별 분리.

## SA가 하는 일

플랫폼 인프라 전체를 구축하고 테넌트에 자원을 배분.

| 업무 | 주요 메뉴 | 문서 |
|------|----------|------|
| 도메인/Zone 구성 | 시스템 자원 > 도메인 (A1001) | [도메인-Zone](../admin-portal/화면/시스템%20자원/01-도메인-Zone.md) |
| 호스트 등록/관리 | 시스템 자원 > 호스트 (A1002) | [호스트](../admin-portal/화면/시스템%20자원/02-호스트-마이그레이션-HA.md) |
| 네트워크 구성 | 시스템 자원 > 네트워크 (A1003) | [네트워크](../admin-portal/화면/시스템%20자원/03-네트워크.md) |
| 스토리지 구성 | 시스템 자원 > 스토리지 (A1005) | [스토리지](../admin-portal/화면/시스템%20자원/04-스토리지.md) |
| 템플릿 정의 | 템플릿 (A09) | [템플릿](../admin-portal/화면/템플릿/01-템플릿.md) |
| 테넌트 생성/할당 | 테넌트 (A08) | [테넌트](../admin-portal/화면/테넌트/01-테넌트.md) |
| 정책 템플릿 정의 | 정책 (A05) | [정책](../admin-portal/화면/정책/) |
| 보안 인증서 | 서비스 > 보안 인증서 (A0603) | [보안 인증서](../admin-portal/화면/서비스/01-보안%20인증서.md) |
| 관리자 UI 커스터마이징 | 포털 > 관리자 UI (A0401) | [관리자 UI](../admin-portal/화면/포털/01-관리자%20UI.md) |
| 관리자 계정/RBAC | 관리자 (A12) | [관리자](../admin-portal/화면/관리자/) |
| Proxy/이메일/HA | 시스템 자원 (A10) | [Proxy-이메일](../admin-portal/화면/시스템%20자원/05-Proxy-이메일.md) |

## TA가 하는 일

SA가 할당한 자원 범위 내에서 테넌트를 운영하고 사용자에게 가상 PC를 제공.

| 업무 | 주요 메뉴 | 문서 |
|------|----------|------|
| 가상 PC 풀/그룹 관리 | 가상 PC (T07) | [가상 PC](../admin-portal/화면/가상%20PC/) |
| 사용자 계정 관리 | 사용자 정보 (T02) | [사용자 정보](../admin-portal/화면/사용자%20정보/) |
| 사용자 지원/원격 제어 | 사용자 지원 (T03) | [사용자 지원](../admin-portal/화면/사용자%20지원/) |
| 정책 설정 | 정책 (T05) | [정책](../admin-portal/화면/정책/) |
| 외부 연동 (AD/NAS/OCTATCO) | 서비스 (T06) | [서비스](../admin-portal/화면/서비스/02-서비스.md) |
| user-portal 콘텐츠 | 포털 (T04) | [포털](../admin-portal/화면/포털/) |
| 테넌트 설정 조정 | 테넌트 (T08) | [테넌트](../admin-portal/화면/테넌트/01-테넌트.md) |
| 가상 디스크 관리 | 가상 디스크 | [가상 디스크](../admin-portal/화면/가상%20디스크/01-가상%20디스크.md) |
| 관리자 계정/권한 그룹 | 관리자 (T12) | [관리자](../admin-portal/화면/관리자/) |
| 자원 현황 조회 | 시스템 자원 (T10) | [시스템 자원](../admin-portal/화면/시스템%20자원/) |
