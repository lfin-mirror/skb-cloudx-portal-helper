# 테넌트 초기 구축 유저플로우

새 조직(회사/부서)에 VDI 환경을 처음 구축할 때의 전체 단계. SA가 인프라를 준비하고, TA가 사용자·VM 환경을 구성.

---

## 전체 구축 순서

```
SA 영역 (인프라 준비)                          TA 영역 (테넌트 운영)
──────────────────                            ──────────────────

[1] 도메인 확인                                      │
     ▼                                              │
[2] 테넌트 생성                                      │
     ▼                                              │
[3] Zone / 호스트 할당                                │
     ▼                                              │
[4] 네트워크 할당                                     │
     ▼                                              │
[5] 스토리지 할당                                     │
     ▼                                              │
[6] 템플릿 준비                                      │
     ▼                                              │
[7] 정책 템플릿 정의                                  │
     ▼                                              │
────── SA → TA 인계 ──────────────────────────       │
                                                     ▼
                                              [8] 정책 가져오기·수정
                                                     ▼
                                              [9] 사용자 그룹 생성
                                                     ▼
                                              [10] 사용자 등록
                                                     ▼
                                              [11] 가상 PC 그룹·풀 생성
                                                     ▼
                                              [12] VM 생성·할당
                                                     ▼
                                              구축 완료 — 사용자 접속 가능
```

---

## SA 영역 — 인프라 준비

### [1] 도메인 확인

```
[SA] 도메인 (A1001)
  │  기존 도메인 확인 (읽기 전용)
  │  도메인 = OpenStack Keystone Domain
  │  일반적으로 사전 생성되어 있음
```

| 화면 | 동작 |
|------|------|
| [도메인](../admin-portal/화면/시스템%20자원/01-도메인-Zone.md#도메인-a1001) (A1001) | 조회만 — 도메인명, 소속 테넌트 목록 |

### [2] 테넌트 생성

```
[SA] 테넌트 (A0802)
  │  "새로운 등록" 클릭
  │  ├── 테넌트명 입력
  │  ├── URL ID 입력 (user-portal 접속 경로)
  │  ├── 호스트 설정 (Y/N) 결정
  │  │     Y: 전용 호스트 모드 → [3]에서 호스트 직접 지정
  │  │     N: Zone 공유 모드 → [3]에서 Zone 선택
  │  ├── 가상 PC 라이선스 수량 설정
  │  └── 저장
  ▼
OpenStack에 Project(테넌트) 생성
```

| 화면 | 핵심 설정 |
|------|----------|
| [테넌트](../admin-portal/화면/테넌트/01-테넌트.md) (A0802) | `host_tnt_yn`, `tnt_nm`, `tnt_url_id`, 라이선스 수량 |

`host_tnt_yn` 선택이 이후 모든 리소스 할당 방식을 결정하는 분기점.

### [3] Zone / 호스트 할당

테넌트에 물리 자원(컴퓨팅) 할당.

```
host_tnt_yn = 'Y' (호스트 사용):
  [SA] 테넌트 상세 → 호스트 설정 버튼
    │  미할당 호스트 목록에서 선택
    │  POST /v1/resource/tenants/manager/usehost
    ▼
  테넌트에 호스트 직접 귀속

host_tnt_yn = 'N' (호스트 미사용):
  [SA] 테넌트 상세 → Zone 추가 버튼
    │  기존 Zone 목록에서 선택
    │  POST /v1/resource/tenants/manager/nonehost
    ▼
  테넌트에 Zone 연결 (호스트는 플랫폼이 자동 배치)
```

필요 시 Zone 신규 생성:

| 화면 | 동작 |
|------|------|
| [Zone](../admin-portal/화면/시스템%20자원/01-도메인-Zone.md#zone-a1002-t1002) (A1002) | Zone 생성 → 호스트 배치 |
| [호스트](../admin-portal/화면/시스템%20자원/02-호스트-마이그레이션-HA.md) (A1003) | 호스트 현황 조회 |

### [4] 네트워크 할당

테넌트에 가상 네트워크 자원 할당.

```
[SA] 테넌트 상세 → 네트워크 관리 섹션
  │  "네트워크 자원 설정" 버튼
  │  물리 네트워크/서브넷 할당
  ▼
또는 직접 네트워크 생성:
  [SA] 네트워크 (A1004)
    │  가상 네트워크 생성 (POST /v1/resource/networks)
    │  서브넷 생성 (POST /v1/resource/subnets)
    │  라우터 생성·연결 (POST /v1/resource/router)
```

| 화면 | 동작 |
|------|------|
| [테넌트](../admin-portal/화면/테넌트/01-테넌트.md) (A0802) | 네트워크 자원 설정 버튼 (SA 전용) |
| [네트워크](../admin-portal/화면/시스템%20자원/03-네트워크.md) (A1004) | 네트워크/서브넷/라우터 CRUD |

### [5] 스토리지 할당

테넌트에 볼륨 타입 할당.

```
[SA] 테넌트 상세 → 볼륨 타입 설정 섹션
  │  "볼륨 타입 선택" 버튼
  │  사용 가능한 볼륨 타입 선택·해제
  ▼
테넌트가 사용할 수 있는 스토리지 프로필 결정
```

| 화면 | 동작 |
|------|------|
| [테넌트](../admin-portal/화면/테넌트/01-테넌트.md) (A0802) | 볼륨 타입 선택/해제 (SA 전용) |
| [스토리지](../admin-portal/화면/시스템%20자원/04-스토리지.md) (A1005) | 볼륨 타입/QoS 관리 |

### [6] 템플릿 준비

VM 스펙 정의. 테넌트 생성 전에 사전 준비되어 있는 경우가 많음.

```
[SA] 베이스 이미지 등록 (A0903)
  │  OS 이미지 등록 (또는 기존 이미지 사용)
  ▼
[SA] 가상 PC 사양 생성 (A0902)
  │  Flavor 정의 (또는 기존 사양 사용)
  ▼
[SA] 템플릿 생성 (A0901)
  │  Flavor + Image 조합
  ▼
[SA] 테넌트 상세 → 템플릿 관리 섹션
  │  "템플릿 설정" 버튼으로 테넌트에 템플릿 할당
```

| 화면 | 동작 |
|------|------|
| [템플릿](../admin-portal/화면/템플릿/01-템플릿.md) (A0901~A0903) | Flavor/Image/템플릿 CRUD |
| [테넌트](../admin-portal/화면/테넌트/01-테넌트.md) (A0802) | 템플릿 설정 버튼 (SA 전용) |

### [7] 정책 템플릿 정의

SA가 정책 기본값을 설정. TA가 이후 복사하여 사용.

```
[SA] 인증 정책 (A0501/A0502) — 로그인 방식, 2차 인증, 비밀번호 규칙
[SA] 가상 PC 정책 (A0503) — 보안 항목, 워터마크, 세션 만료 등
[SA] 백업/스냅샷 정책 (A0505) — 백업 주기, 보관 기간
```

| 화면 |
|------|
| [인증 정책](../admin-portal/화면/정책/01-인증%20정책.md) (A0501, A0502) |
| [가상 PC 정책](../admin-portal/화면/정책/02-가상%20PC%20정책.md) (A0503) |
| [백업/스냅샷 정책](../admin-portal/화면/정책/03-백업%20스냅샷%20정책.md) (A0505) |

## TA 영역 — 테넌트 운영 구성

SA가 인프라를 준비한 후, TA가 테넌트 내부를 구성.

### [8] 정책 가져오기·수정

```
[TA] 인증 정책 (T0501/T0502)
  │  "가져오기" → SA 템플릿 복사
  │  테넌트 환경에 맞게 수정
  ▼
[TA] 가상 PC 정책 (T0503)
  │  "가져오기" → SA 템플릿 복사
  │  상세 정책 먼저 생성 (예외 네트워크, USB, 보안 그룹 등)
  │  가상 PC 정책에서 상세 정책 연결
  ▼
[TA] 백업/스냅샷 정책 (T0505)
  │  "가져오기" → SA 템플릿 복사
  │  백업 주기/보관 기간 조정
```

정책 흐름 상세: [06-policy-flow.md](./06-policy-flow.md) 참고.

### [9] 사용자 그룹 생성

```
[TA] 사용자 그룹 (T0201)
  │  부서/팀 단위 그룹 생성
  │  그룹별 인증 정책 연결
  │  AD 연동 시 AD 그룹 동기화
```

| 화면 | 설정 |
|------|------|
| [사용자 그룹](../admin-portal/화면/사용자%20정보/01-사용자%20그룹.md) (T0201) | 그룹명, 인증 정책, AD 동기화 |

### [10] 사용자 등록

```
[TA] 사용자 (T0202)
  │  개별 등록 or 일괄 등록 (엑셀)
  │  사용자 그룹 배정
  │  AD 연동 시 AD 계정 동기화
```

| 화면 | 설정 |
|------|------|
| [사용자](../admin-portal/화면/사용자%20정보/02-사용자.md) (T0202) | 계정 ID, 이름, 그룹, AD 연동 여부 |

### [11] 가상 PC 그룹·풀 생성

```
[TA] 가상 PC 그룹 생성 (T0701)
  │  ├── 템플릿 선택 (SA가 할당한 템플릿 중 선택)
  │  ├── 네트워크 설정 (할당된 네트워크/서브넷 중 선택)
  │  ├── 가상 PC 정책 선택 ([8]에서 생성한 정책)
  │  └── 백업/스냅샷 정책 선택 ([8]에서 생성한 정책)
  ▼
[TA] 풀 생성
  │  ├── Zone 선택
  │  ├── 전용/공용 PC 유형 결정
  │  ├── 볼륨 타입 선택
  │  └── 전원 정책 / 초기화 정책
```

| 화면 | 설정 |
|------|------|
| [가상 PC 그룹](../admin-portal/화면/가상%20PC/01-가상%20PC%20그룹.md) (T0701) | 템플릿, 네트워크, 정책 |
| 풀 상세 (T0701 하위) | Zone, 볼륨 타입, 전용/공용, 전원 정책 |

### [12] VM 생성·할당

```
[TA] 가상 PC 그룹 목록 → "생성" 버튼
  │  VM 수량 입력 → 프로비저닝
  ▼
[TA] 사용자에게 VM 할당
  │  가상 PC (T0702) → 사용자 선택
  │  또는 자동 할당 맵핑 (T0707)으로 자동화
  ▼
구축 완료 — 사용자가 user-portal에서 접속 가능
```

| 화면 | 동작 |
|------|------|
| [가상 PC 그룹](../admin-portal/화면/가상%20PC/01-가상%20PC%20그룹.md) (T0701) | VM 생성 |
| [가상 PC](../admin-portal/화면/가상%20PC/02-가상%20PC.md) (T0702) | VM 할당 |

## 전체 의존 관계

```
[SA 인프라]
도메인 (A1001) → 테넌트 (A0802)
                    │
          ┌─────────┼──────────┐
          ▼         ▼          ▼
     Zone/호스트  네트워크   스토리지
     (A1002)     (A1004)    (A1005)
          │         │          │
          └─────────┼──────────┘
                    ▼
              템플릿 (A0901)
              정책 (A0503, A0505)
                    │
         ─── SA → TA 인계 ───
                    │
[TA 운영]           ▼
              정책 가져오기 (T0503, T0505)
                    │
          ┌─────────┼──────────┐
          ▼         ▼          ▼
     사용자 그룹  사용자     가상 PC 그룹
     (T0201)    (T0202)    (T0701)
                    │          │
                    └────┬─────┘
                         ▼
                    VM 생성·할당
                    (T0701, T0702)
                         │
                         ▼
                    사용자 접속 가능
```
