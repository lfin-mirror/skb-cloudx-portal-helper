# 추측 문서 개요

코드 분석과 실제 환경 데이터를 기반으로 작성한 추측 문서. 공식 문서나 구축 가이드가 없는 영역을 코드에서 역추적하여 정리. 코드 검증된 부분과 추측 부분을 명시적으로 구분.

## 문서 목록

| 문서 | 핵심 질문 | 검증 수준 |
|------|----------|----------|
| [01-초기 구축 흐름 추측](01-초기-구축-흐름-추측.md) | "OpenStack 인프라부터 사용자가 VM을 쓰기까지 어떤 순서로 구축하는가?" | 8단계 흐름 중 4~7단계는 코드 검증, 1~3단계는 추측 |
| [02-VDI 클라이언트 접속 흐름 추측](02-vdi-클라이언트-접속-흐름-추측.md) | "Cloud PC 실행 버튼을 누르면 어떤 일이 일어나는가?" | 포털 코드(URI 구성, 검증 로직, 브라우저 분기)는 검증 완료, 클라이언트 앱 내부 동작은 추측 |

## 읽기 순서

`01-초기 구축 흐름` → `02-VDI 클라이언트 접속 흐름` 순서 권장. 01에서 전체 구축 흐름의 끝이 "사용자가 VDI 접속"이고, 02가 그 접속 과정을 상세하게 다룸.

## 추측 표기 규칙

- **(코드 검증)** — 소스코드에서 직접 확인한 내용
- **(추측)** — 코드 흐름과 도메인 지식 기반 추론. 실제 동작과 다를 수 있음
- **미확인 사항** — 각 문서 하단에 아직 검증하지 못한 항목 정리
