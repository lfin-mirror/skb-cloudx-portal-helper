# 전용 PC / 공용 PC

CloudX VDI에서 VM(가상 PC)을 사용자에게 할당하는 두 가지 방식.

## 코드

| 코드 | 값 | 의미 |
|------|-----|------|
| `tnt_mtd_cd` | `V001DED` | 전용 PC (Dedicated) |
| `tnt_mtd_cd` | `V001POO` | 공용 PC (Pooled) |

## 전용 PC (`V001DED`)

사용자 1명에 VM 1대가 **고정 할당**. 사용자가 로그아웃해도 VM이 유지되고, 다음 로그인 시 같은 VM에 접속.

```
사용자 A → VM-001 (항상 같은 VM)
사용자 B → VM-002 (항상 같은 VM)
사용자 C → VM-003 (항상 같은 VM)
```

- 비유: 회사에서 내 자리에 있는 **내 PC**. 퇴근해도 내 PC는 그대로.
- IP 할당: 고정 IP(`P006S`) 또는 동적 IP(`P006D`) 선택 가능.
- VM 내 개인 설정, 파일, 프로그램이 유지됨.

## 공용 PC (`V001POO`)

사용자가 로그인할 때 **대기 중인 VM 풀에서 하나를 자동 배정**. 로그아웃하면 VM이 풀로 반환.

```
사용자 A 로그인  → 풀에서 VM-001 배정
사용자 A 로그아웃 → VM-001 풀로 반환
사용자 D 로그인  → 풀에서 VM-001 배정 (같은 VM을 다른 사용자가 쓸 수 있음)
```

- 비유: 도서관 **공용 PC**. 자리에 앉으면 쓰고, 일어나면 다른 사람이 쓸 수 있음.
- IP 할당: 동적 IP(`P006D`) 고정 — 흐름 방식.
- VM이 반환될 때 초기화되므로 개인 데이터 유지 안 됨 (개인 디스크로 별도 관리).

## 비교

| 항목 | 전용 PC (`V001DED`) | 공용 PC (`V001POO`) |
|------|-------------------|-------------------|
| VM 할당 | 사용자에게 고정 | 풀에서 동적 배정 |
| 로그아웃 시 VM | 유지 (전원만 꺼짐) | 풀로 반환 (다른 사용자에게 재배정 가능) |
| IP 할당 방식 | 고정(`P006S`) 또는 동적(`P006D`) 선택 | 동적(`P006D`) 고정 |
| 개인 데이터 | VM 내 유지 | 초기화됨 → 개인 디스크(가상 디스크)로 관리 |
| VM 효율 | 사용자 수 = VM 수 | 동시 접속자 수 < 사용자 수 가능 (자원 절약) |
| 적합 환경 | 개인화 필요 (개발자, 디자이너) | 표준화된 업무 (콜센터, 교대 근무) |

## admin-portal에서의 사용

### 가상 PC 그룹 (T0701) → 풀 생성 시

풀(Pool) 단위로 전용/공용 결정. 하나의 그룹 안에 전용 풀과 공용 풀이 공존 가능.

### 가상 PC (T0702) 목록

체크박스 필터로 전용/공용 구분 조회. Pool 드롭다운에서 아이콘으로 시각적 구분.

### 가상 PC 예약 (T0704)

탭으로 전용/공용 구분. 할당 준비/실패 상태 모니터링.

### 공용 PC 대기 (T0706)

공용 PC 전용 메뉴. 풀에 반환된 대기 상태의 공용 PC 관리 + 자원 회수.
