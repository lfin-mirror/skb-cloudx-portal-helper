# 문서 작성 공통 규칙

## 파일명

- 파일명 앞에 넘버링 필수: `01-`, `02-`, `03-`, ...
- 넘버링은 각 폴더 내부에서 순차 적용 (폴더가 다르면 각각 01부터 시작)
- 파일 내용이 변경되어 기존 파일명이 내용을 대표하지 못하면 파일명도 함께 변경. 파일명 변경 시 해당 파일을 참조하는 모든 링크도 함께 업데이트.

## 중복 금지

- 파일 간 내용 중복 불허.
- 이미 다른 파일에 작성된 내용을 다시 써야 하는 상황이 생기면, 작성 전에 반드시 AskUserQuestion으로 사용자에게 확인:
  - 선택지 1: 중복을 허용하고 그대로 작성
  - 선택지 2: 중복되는 내용을 별도 파일로 분리하고 양쪽에서 링크
- 사용자 확인 없이 중복 내용을 작성하지 않음.

## 문체

- 문장을 동사로 끝내지 않음. 명사/단어로 끝냄.
  - O: `sessionStorage에 토큰 저장.`
  - X: `sessionStorage에 토큰을 저장한다.`
- 표 안의 셀도 동일하게 명사/단어로 끝냄.

## 흐름 문서

- 화면 이동 흐름 문서의 파일명: `{시작점}-to-{도착점}-flow.md`
- 시작점이 메인. 도착점 상세는 해당 도착점이 시작점인 다른 문서에서 기술.

## 작업 디렉토리

- Bash에서 `cd`로 경로를 이동한 경우, 작업이 끝나면 반드시 `/Users/jay/skb/cloudx`로 복귀.
